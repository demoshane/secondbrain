# Phase 36: Chrome Extension — Web & Email Capture

## Objective

Build a Chrome Manifest V3 extension that captures web pages, selected text, and Gmail email threads into the second brain via the existing `sb-api` HTTP endpoint.

## Why

95% of captures come through MCP, but high-value web content (articles, Gmail threads, reference pages) requires copy-paste into Claude. A browser-native capture path removes that friction entirely.

## Architecture

```
Chrome Extension (Manifest V3)
  ├── popup.html/js    — edit before save UI
  ├── background.js    — service worker, context menu
  ├── content.js       — page content extraction
  └── gmail.js         — Gmail-specific thread extraction
        ↓ fetch()
  sb-api (localhost:37491)
    POST /smart-capture   { content, source_url, source_type }
    POST /capture          { note_type, title, body, tags, ... }
        ↓
  capture_note() → ~/SecondBrain/*.md
```

## Sub-phases

### 36-01: Extension scaffold + basic page capture
- Manifest V3 setup (permissions: activeTab, contextMenus, storage)
- Context menu: "Capture to Second Brain" on right-click
- Content script: extract page title + selected text (or full readable content via Readability.js)
- Popup UI: shows extracted content, allows edit title/body/tags before save
- POST to `localhost:37491/smart-capture`
- Connection status indicator (sb-api up/down)
- CORS: add `chrome-extension://*` to Flask CORS origins

### 36-02: Gmail thread extraction
- Content script for `mail.google.com` — detect email view
- Extract: sender, recipients, subject, date, thread body (multi-message)
- Auto-classify as `meeting` (if calendar invite), `note` (general), `decision` (if action items detected)
- Preserve Gmail URL as source link in frontmatter
- "Capture this email" button injected into Gmail UI (or context menu only — TBD)

### 36-03: Rich capture UX
- Keyboard shortcut (Cmd+Shift+B) for quick capture
- Badge count showing recent captures
- Capture history in popup (last 10, from localStorage)
- Options page: configure sb-api URL, default tags, auto-capture behavior
- Toast notification on successful capture

## Backend changes needed

1. **CORS**: add `chrome-extension://*` to `flask_cors` origins in `api.py`
2. **`POST /smart-capture`**: accept optional `source_url` and `source_type` fields → store in frontmatter
3. **`POST /capture`**: accept optional `source_url` field → store in frontmatter
4. **Health endpoint**: `GET /ping` (lightweight, for extension connection check)

## Tech stack

- Manifest V3 (Chrome 88+)
- Vanilla JS + minimal CSS (no framework — keep it tiny)
- Mozilla Readability.js (BSD, ~15KB) for article extraction
- DOMPurify for sanitizing extracted HTML before sending

## Security considerations

- Extension only talks to localhost — no external network calls
- DOMPurify sanitizes all extracted HTML before capture
- No credentials stored in extension (sb-api has no auth on localhost)
- `host_permissions` limited to `http://localhost:37491/*` and `*://mail.google.com/*`

## Out of scope (future phases)

- Firefox/Safari port
- Floating macOS capture widget (screenshot + OCR from any app)
- Cloud sync / remote sb-api (would need auth)
- Auto-capture (capture without user interaction)

## Done criteria

- Can right-click any page → capture selected text or full article
- Can capture Gmail email thread with sender/subject/date preserved
- Cmd+Shift+B opens quick capture popup
- All captures land in ~/SecondBrain with correct type classification
- Extension shows connection status to sb-api
- Installable from `chrome://extensions` (developer mode, load unpacked)
