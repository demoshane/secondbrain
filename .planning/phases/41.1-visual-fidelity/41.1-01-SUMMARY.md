## What Was Built

### Task 1 — HealthScoreGauge SVG Arc

Replaced the plain `<span>` number display with a full SVG circular arc gauge:
- `viewBox="0 0 120 120"`, circle at (60,60) radius 50, `w-24 h-24`
- Background track circle using `hsl(var(--border))` stroke
- Foreground arc using `strokeDasharray` (circumference=~314.16) and `strokeDashoffset` (circumference × (1 - score/100))
- `strokeLinecap="round"` for polished arc ends
- `transform="rotate(-90 60 60)"` starts arc at 12 o'clock
- Color thresholds: `#22c55e` (green-500, >=80), `#fbbf24` (amber-400, >=50), `#ef4444` (red-500, <50)
- Centered `<text>` element with `textAnchor="middle"` `dominantBaseline="central"` shows the score number in matching color
- "Brain Health" label span below SVG preserved
- Export name and props interface unchanged

### Task 2 — Sidebar Two-Level Folder+Type Grouping

Replaced `groupByType()` with `groupByFolderThenType()`:
- Extracts folder from first path segment: `note.path.indexOf('/')` — no `/` → `'other'`
- `FOLDER_ORDER` array for deterministic folder ordering; unknowns appended alphabetically
- `TYPE_ORDER` preserved for type ordering within each folder
- Outer `CollapsibleSection` per folder: `sectionId="sidebar-folder-{folder}"`, count = total notes in folder
- Inner `CollapsibleSection` per type: `sectionId="sidebar-{folder}-{type}"`, wrapped in `<div className="pl-2">` for indentation
- All existing functionality preserved: tag filter banner, search results mode, active note highlighting, NoteRow unchanged
- Old `groupByType()` function removed

## Deviations from Plan

### Minor Deviation

**[Rule 1 - Implementation detail] SVG stroke color as hex instead of Tailwind class names**
- **Found during:** Task 1 implementation
- **Issue:** SVG `stroke` attribute is an HTML/SVG attribute, not a CSS class. Tailwind class names like `stroke-green-500` would require additional Tailwind SVG stroke plugin or CSS variables. The plan spec said `green-500`, `amber-400`, `red-500` as color references.
- **Fix:** Used the equivalent hex values (`#22c55e`, `#fbbf24`, `#ef4444`) directly on the `stroke` attribute. Color semantics are identical.
