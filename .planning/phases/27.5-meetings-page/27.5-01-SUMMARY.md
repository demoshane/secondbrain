## What Was Built

**GET /meetings** — Returns `{"meetings": [...]}` where each item has:
- `path`: absolute filesystem path
- `title`: note title
- `meeting_date`: ISO date string from `substr(created_at,1,10)`
- `participant_count`: integer count of `json.loads(people)` array length
- `open_actions`: correlated subquery count from `action_items WHERE note_path=n.path AND done=0`

**GET /meetings/<path>** — Returns full meeting detail:
- Same fields as list plus `body` (from DB) and `participants` (array instead of count)
- Path-traversal guard via `_resolve_note_path()` — returns 403 on escape attempt
- Returns 404 when note not in DB

**list_actions() note_path param** — Added `note_path: str | None = None` parameter with conditional SQL `AND note_path=?`. Exposed as `?note_path=` on GET /actions.
