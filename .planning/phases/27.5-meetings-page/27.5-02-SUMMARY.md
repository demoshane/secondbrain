## What Was Built

### MeetingsPage.tsx
Split-panel layout mirroring PeoplePage.tsx exactly:

- **Left panel** (w-80): filterable table with columns Title / People / Date / Actions (open count). Sort is date-desc from API — no client-side sort needed. Empty state: "No meetings found".
- **Right panel**: Shows "Select a meeting to view details" until a row is clicked, then renders four collapsible sections:
  1. **Note** — full body via react-markdown + remark-gfm
  2. **Participants** — bullet list from `participants` string array (`data-testid="participants-section"`)
  3. **Action Items** — open actions from `GET /actions?note_path=<enc>` (`data-testid="meeting-actions-section"`)
  4. **Backlinks** — from `GET /notes/<enc>/meta` (`data-testid="meeting-backlinks-section"`)
- "Open in Notes" button calls `openNote(selectedPath)` then `setCurrentView('notes')`
- Outer div has `data-testid="meetings-page"`

### Wiring changes
- `types.ts`: Added `MeetingSummary` interface (path, title, meeting_date, participant_count, open_actions)
- `UIContext.tsx`: Extended `View` union to `'notes' | 'actions' | 'people' | 'meetings'`
- `TabBar.tsx`: Added `{ id: 'meetings' as const, label: 'Meetings' }` after people entry
- `App.tsx`: Added import, changed final else branch to `currentView === 'people' ? <PeoplePage /> : <MeetingsPage />`
