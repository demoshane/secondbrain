## What Was Built

### Task 1 — Tab Data-Correctness Tests (test_gui.py)

Four new test functions appended that assert real seeded content appears in each tab, not just that a container element exists:

- `test_notes_tab_shows_real_content` — seeds "Notes Smoke Note", waits for sidebar, asserts first note-item has non-empty inner text
- `test_people_tab_shows_real_content` — navigates to People tab, asserts "Test Person" row visible in tbody
- `test_meetings_tab_shows_real_content` — navigates to Meetings tab, asserts "Q1 Kickoff" row visible
- `test_projects_tab_shows_real_content` — navigates to Projects tab, asserts "Test Project" row visible

All 4 use the `gui_brain` session-scoped fixture which seeds the required test data.

### Task 2 — Orphan Detection Spec Tests (test_brain_health.py)

Three new spec tests in the `# Orphan detection correctness spec` section using isolated in-memory SQLite DBs:

- `test_orphan_is_note_with_no_relationships` — note absent from both source/target columns is an orphan
- `test_note_with_outbound_link_is_not_orphan` — note as source_path is not an orphan
- `test_note_with_inbound_link_is_not_orphan` — note as target_path is not an orphan

Each test patches `engine.db.DB_PATH` and `engine.paths.DB_PATH` to `tmp_path` for full isolation, following the existing LEARNINGS.md isolation rule.
