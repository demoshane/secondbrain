**Test:** Use the Status dropdown (open/done), Assignee dropdown, and Show done checkbox
**Expected:** Each control re-fetches and re-renders the table with the appropriate subset of items
**Why human:** JavaScript event wiring and re-render require browser interaction

#### 4. Assignee Picker — Single Click

**Test:** Click any Assignee cell (even an empty "—" cell) in the Actions table
**Expected:** A dropdown opens immediately (one click) listing all people notes; selecting one updates the cell and persists; clicking elsewhere dismisses without change
**Why human:** showPicker() and DOM positioning require a browser; the Plan 04 bug fix must be confirmed working

#### 5. My Actions on Person Note

**Test:** Click "Notes" tab, open a note that has type: people in its frontmatter
**Expected:** Right meta panel shows a "My Actions" section listing open items assigned to that person; if none assigned, shows "None assigned"; opening a non-person note hides the section entirely
**Why human:** Conditional section and async fetch require interaction with actual notes

#### 6. Done Checkbox Reopen

**Test:** In the Action Items table, show done items (check "Show done"); uncheck a done item's checkbox
**Expected:** Item disappears from the done view, reappears when viewing open items — the Plan 04 reopen bug fix must be working
**Why human:** State toggle flow requires browser interaction to confirm correct before/after behavior
