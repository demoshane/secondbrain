# Phase 27.2: Action Items Page + Nav Scaffold - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the single-panel GUI with a top-level tab bar and deliver a dedicated Action Items page with a table layout, assignee support, and filtering. Person notes get a "My Actions" section in the right sidebar meta panel. The Notes tab preserves existing functionality but hides the right panels (viewer-focused layout).

</domain>

<decisions>
## Implementation Decisions

### Tab bar placement & layout
- **Top-level nav**: tab bar sits below the topbar, above the layout
- When on the **Notes tab**: sidebar + viewer only — right panels hidden (viewer-focused, more space)
- When on the **Action Items tab**: sidebar + full-width Actions page (no right panel)
- **Active tab style**: background highlight (filled/pill) on active tab
- **Extensible**: tabs defined as a data-driven array `[{id, label}]` — adding a future page = appending one entry, no structural HTML changes

### Action Items page design
- **Table layout** with columns: Done (checkbox) | Task text | Source note | Assignee | Created
- **Default view**: open items only; a "Show done" toggle reveals completed items
- **Filter row** above the table: [Assignee: all ▾] [Status: open ▾] dropdowns

### Assignee schema & picker
- **DB migration**: add two columns to `action_items`: `assignee_path TEXT NULL` and `due_date TEXT NULL` (ISO date string)
- **Inline click-to-assign**: clicking the Assignee cell in a table row shows a small dropdown of all `type=people` notes; selecting one writes `assignee_path`
- **People options source**: `GET /notes?type=people` — consistent with how person chips work

### My Actions in person note
- **Right sidebar meta panel**: a new section below the existing Backlinks and People sections
- Shows all open action items where `assignee_path` matches the current note's path
- **Trigger condition**: only shown when the open note has `type: people` in its frontmatter

### Claude's Discretion
- Exact CSS for tab bar (height, padding, font size)
- Whether to add a due_date column to the table display in this phase or just store it in DB
- Exact API endpoint design for the new filter params (`GET /actions?done=0&assignee=...`)
- My Actions section header copy ("My Actions" vs "Assigned Actions" vs "Open Actions")

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `loadActions()` in `app.js`: existing fetch + render cycle for action items panel — reuse the pattern, extend to full-page table
- `POST /actions/<id>/done` endpoint: already exists, reuse for checkbox in new table
- `GET /actions` endpoint: returns open items — extend with `?done=0/1&assignee=path` query params
- `notes` table `people` column: JSON array — already used for person chips in meta panel
- Existing `<section>/<ul>` pattern in right panel: My Actions section follows the same structure

### Established Patterns
- All list rendering: `innerHTML = items.map(i => \`<li>...</li>\`).join('')` — new table follows the same fetch-then-innerHTML cycle
- Vanilla JS only, no framework — new tab bar and page are DOM manipulation + event listeners
- `migrate_add_*` pattern in `engine/db.py` — use this pattern for `assignee_path` + `due_date` migration
- Section structure: `<section id="xxx-panel"><h3>Title</h3>...</section>`

### Integration Points
- Tab bar inserts between `<header id="topbar">` and `<div id="layout">` in `index.html`
- `showView('notes'|'actions')` state variable controls which view is visible
- Notes tab hides `#right-panel` (new behavior) and shows `#layout` as-is but without right panel
- `loadMeta()` in `app.js` already fires when a note opens — hook My Actions section here, conditioned on `note.type === 'people'`
- `GET /notes/<path>/meta` already returns note metadata — add `assignee_path` check here or via new `GET /actions?assignee=<path>`

</code_context>
