<objective>
Replace the fragile hardcoded-Homebrew-path pre-commit hook with a portable .githooks/pre-commit wrapper versioned in the repo, and update devcontainer.json to use git config core.hooksPath instead of pre-commit install.

Purpose: UAT test 7 (major) — .git/hooks/pre-commit hardcodes /usr/local/opt/pre-commit/libexec/bin/python3.14 (Homebrew macOS path). Container's postCreateCommand runs `uv run pre-commit install` which overwrites the host hook; host's `pre-commit install` overwrites it back. Inside container the Homebrew path doesn't exist and bare `pre-commit` isn't on PATH, so commits fail.

Fix: Create .githooks/pre-commit wrapper that calls `uv run pre-commit run --hook-stage pre-commit -- "$@"`. Set git core.hooksPath to .githooks in devcontainer.json postCreateCommand. Remove `uv run pre-commit install` from postCreateCommand. Host users run `git config core.hooksPath .githooks` once (document in README).
Output: .githooks/pre-commit (executable, portable), devcontainer.json updated.
</objective>

    # Portable pre-commit wrapper.
    # Works on macOS host (uv in PATH) and inside DevContainer (uv in venv).
    # Activate via: git config core.hooksPath .githooks
    # (DevContainer sets this automatically in postCreateCommand.)

    set -e

    if command -v uv >/dev/null 2>&1; then
        exec uv run pre-commit run --hook-stage pre-commit --color always "$@"
    else
        echo "[pre-commit] ERROR: uv not found. Install uv: https://docs.astral.sh/uv/getting-started/installation/"
        exit 1
    fi
