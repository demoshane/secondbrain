<task type="auto" tdd="true">
  <name>Task 3: File upload size cap + thread-safe suppress_next_delete</name>
  <files>engine/api.py, engine/watcher.py</files>
  <behavior>
    api.py upload route:
    - Check Content-Length header before processing: if > 52428800 (50 MB) return 413
    - Also check actual file size after save: if > 50 MB, delete and return 413
    - Add test: upload a mock file > 50 MB → assert 413 response

    watcher.py suppress_next_delete:
    - Replace boolean flag with threading.Event or a dict {path: expiry_ts}
    - suppress_next_delete(path): set _suppressed[path] = time.monotonic() + 2.0
    - is_suppressed(path): return _suppressed.get(path, 0) > time.monotonic(); pop if expired
    - Thread-safe: use threading.Lock for _suppressed dict mutations
  </behavior>
  <action>
    Read engine/api.py upload route; read engine/watcher.py suppress_next_delete;
    apply both fixes; run full suite
