  <files>engine/watcher.py, tests/test_watcher.py</files>
  <action>
    Replace the current set-based `suppress_next_delete` in engine/watcher.py with a per-path Event approach:

    ```python
    _suppress_events: dict[str, threading.Event] = {}
    _suppress_lock = threading.Lock()

    def suppress_next_delete(abs_path: str, window: float = 0.5) -> None:
        ev = threading.Event()
        with _suppress_lock:
            _suppress_events[abs_path] = ev
        threading.Timer(window, _clear_suppress, args=(abs_path,)).start()

    def is_suppressed(abs_path: str) -> bool:
        with _suppress_lock:
            return abs_path in _suppress_events

    def _clear_suppress(abs_path: str) -> None:
        with _suppress_lock:
            _suppress_events.pop(abs_path, None)
    ```

    Update any callers that check the old suppress set to use `is_suppressed()` instead.

    Add tests in test_watcher.py:
    - suppress_next_delete makes is_suppressed return True
    - After window expires, is_suppressed returns False
    - Multiple paths can be suppressed independently
  </action>
