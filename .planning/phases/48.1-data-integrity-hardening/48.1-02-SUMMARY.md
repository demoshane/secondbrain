# 48.1-02 Summary: batch-capture + api.py _NP migration

## Task 1: batch-capture rewrite

Rewrote `POST /batch-capture` to:
- Parse full frontmatter (tags, people, sensitivity, url) — not just title/type/body
- Store all fields in the INSERT so SQLite triggers populate junction tables
- Use relative paths for both notes and attachments (was absolute for attachments)
- Use `relative_to(brain_root)` consistently instead of mixed `resolve()` patterns

Did NOT switch to `capture_note()` per file — that would re-create files on disk
and run entity extraction on every existing note. The endpoint's purpose is to index
existing files, not capture new ones. The trigger-based junction sync makes the
direct INSERT correct.

## Task 2: api.py _NP migration

Migrated 16 route handlers from the old pattern:
```python
p, _brain_root = _resolve_note_path(note_path)
sp = store_path(p)  # manual conversion
```
To the new pattern:
```python
np = _resolve_note_path(note_path)
# np.absolute for disk I/O, np.relative for DB queries
```

**Result:** `store_path()` calls in api.py route handlers dropped from 33 to 6.
The remaining 6 are legitimate boundary conversions on `capture_note()` return
values — not from `_resolve_note_path`.
