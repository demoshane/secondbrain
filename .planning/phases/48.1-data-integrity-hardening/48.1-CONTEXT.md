# Phase 48.1 — Data Integrity Hardening

## Origin
Bug hunt session (April 2026) found ~45 bugs. Most path and error-handling issues
were fixed inline. Two structural improvements require dedicated work:

1. **Junction table dual-write fragility** — `note_tags` and `note_people` must be
   updated alongside JSON columns at 5+ write paths. Missing any one causes search
   filters to silently return stale results. SQLite triggers would make this impossible.

2. **`POST /batch-capture` bypasses `capture_note()`** — The GUI "refresh" button
   does raw SQL INSERTs, skipping entity extraction, junction tables, audit log,
   backlinks, and intelligence hooks. Should use `capture_note()` per file.

## Additional cleanup (low effort, bundle here)
3. Migrate remaining `store_path(p)` calls in api.py routes to use `_NP.relative`
   from `_resolve_note_path()`. The `_NP` class and `__iter__` backward compat are
   already in place — this is pure mechanical find-replace (~26 routes).

## Execution strategy: direct
All plans modify api.py and db.py — no parallelism possible.
