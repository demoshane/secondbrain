# 48.1-01 Summary: SQLite Triggers for Junction Table Sync

## What was done

Added 4 SQLite triggers on the `notes` table that auto-populate `note_tags` and `note_people`
junction tables from the JSON `tags` and `people` columns:

- `notes_tags_ai` — AFTER INSERT, syncs note_tags from notes.tags
- `notes_tags_au` — AFTER UPDATE OF tags, syncs note_tags
- `notes_people_ai` — AFTER INSERT, syncs note_people from notes.people
- `notes_people_au` — AFTER UPDATE OF people, syncs note_people

Then removed all manual dual-write code from:
- `write_note_atomic()` in capture.py (~20 lines)
- `update_note()` in capture.py (~15 lines)
- `save_note` tags-only branch in api.py (~10 lines)
- `save_note` people-only branch in api.py (~10 lines)
- `reindex_brain()` bulk rebuild in reindex.py (~20 lines)

## Impact

The entire bug class "wrote tags JSON but forgot junction table" is now impossible.
Any code that writes to `notes.tags` or `notes.people` (via INSERT or UPDATE)
automatically gets the junction tables synced by the database itself.
