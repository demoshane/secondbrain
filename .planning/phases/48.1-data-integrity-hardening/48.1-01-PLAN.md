<objective>
Replace manual dual-write of note_tags/note_people junction tables with SQLite
AFTER INSERT/UPDATE triggers on the notes table. Eliminates the entire bug class
where a write path updates the JSON column but forgets the junction table.
</objective>

<context>
@engine/db.py — schema + migrations
@engine/capture.py — write_note_atomic, update_note (manual dual-write)
@engine/api.py — save_note tags/people branches (manual dual-write)
@engine/reindex.py — bulk junction rebuild we just added
</context>

<tasks>
<task type="auto" tdd="true">
  <behavior>Triggers auto-populate note_tags from notes.tags JSON on INSERT/UPDATE</behavior>
  <action>
    Add AFTER INSERT and AFTER UPDATE triggers on `notes` that:
    1. DELETE FROM note_tags WHERE note_path = NEW.path
    2. INSERT INTO note_tags SELECT NEW.path, j.value FROM json_each(NEW.tags) AS j WHERE j.value != ''
    Same pattern for note_people from notes.people JSON.
    Add as migration function in db.py, called from init_schema().
  </action>
  <verify><automated>uv run pytest tests/test_path_integrity.py tests/test_capture.py -x -q</automated></verify>
  <done>Junction tables populated by triggers, not application code</done>
</task>

<task type="auto">
  <behavior>Remove manual dual-write code now handled by triggers</behavior>
  <action>
    Remove note_tags/note_people INSERT loops from:
    - write_note_atomic() in capture.py
    - update_note() in capture.py
    - save_note tags-only branch in api.py
    - save_note people-only branch in api.py
    - reindex_brain() bulk rebuild in reindex.py
    Keep the DELETE+INSERT only in places where triggers won't fire (direct conn.execute UPDATE).
  </action>
  <verify><automated>uv run pytest tests/ -x -q</automated></verify>
  <done>Zero manual junction table writes remain</done>
</task>
</tasks>
