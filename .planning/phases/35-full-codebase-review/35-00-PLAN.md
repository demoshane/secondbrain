<objective>
Full codebase audit using /agent-teams:team-review across security, architecture,
performance, and testing dimensions. Produce prioritised findings and break down
into actionable fix plans.
</objective>

<context>
- Codebase: ~38 Python modules (engine/), ~40 React/TS components (frontend/src/)
- 29 phases of accumulated work — never had a cross-cutting review
- Tool: /agent-teams:team-review with parallel reviewers per dimension
- Security standards: customer data must not leak, no destructive infra access,
  Jira/external system writes protected by guard hooks
</context>

<approach>
Wave 1: Audit
- Run /agent-teams:team-review on engine/*.py — security + architecture + performance
- Run /agent-teams:team-review on frontend/src/**/*.tsx — security + architecture + performance
- Run /agent-teams:team-review on tests/ — coverage gaps + test quality

Wave 2: Triage
- Consolidate findings across all three reviews
- Deduplicate and severity-rank (Critical → Low)
- Create individual fix plans for Critical and High findings
- Document Medium/Low as tech debt backlog

Wave 3: Remediation
- Execute fix plans (one plan per Critical/High finding or grouped by area)
- Re-run team-review on changed files to verify fixes
- Update SECURITY.md if security findings are resolved

Success criteria:
- Zero Critical findings remaining
- All High findings resolved or have tracked mitigation plan
- Medium/Low documented in tech debt backlog
- Test coverage gaps filled for critical paths identified in review
</approach>
