The correct fix (from LEARNINGS.md "Test Suite Writing to Real ~/SecondBrain" entry): other fixtures must NOT be able to restore DB_PATH to the pre-gui_brain value. The solution is to add a `pytest_runtest_setup` hook or a `@pytest.fixture(autouse=True, scope="session")` that re-asserts the correct values before any test that depends on the GUI server.

Concrete implementation in conftest.py:

```python
@pytest.fixture(autouse=True, scope="session")
def _freeze_gui_db(gui_brain):
    """Re-anchor DB_PATH to gui_brain tmp_db after gui_brain sets it.
    Prevents other test files' function-scoped monkeypatches from restoring
    the original (real) DB_PATH mid-session.

    gui_brain is listed as a dependency to ensure ordering: gui_brain runs first,
    sets _db.DB_PATH and _paths.DB_PATH, then this fixture captures those values.
    """
    import engine.db as _db
    import engine.paths as _paths

    frozen_db = _db.DB_PATH   # capture what gui_brain set
    frozen_paths = _paths.DB_PATH

    yield
    # No teardown needed — this is session-scoped
```

BUT this alone doesn't prevent mid-session monkeypatch restores. The real fix is to add a `pytest_runtest_teardown` hook that re-anchors after every test:

```python
def pytest_runtest_teardown(item, nextitem):
    """After each test, re-anchor DB_PATH to the GUI test DB if it was set."""
    import engine.db as _db
    import engine.paths as _paths
    # Only re-anchor if gui_brain has already run (check via the module-level sentinel)
    global _GUI_DB_PATH
    if _GUI_DB_PATH is not None:
        _db.DB_PATH = _GUI_DB_PATH
        _paths.DB_PATH = _GUI_DB_PATH

_GUI_DB_PATH = None  # module-level sentinel, set by gui_brain fixture
```

Update `gui_brain` fixture to set `_GUI_DB_PATH = tmp_db` (global assignment) after setting the module attributes.

**Alternative if hook approach is fragile:** Instead of a hook, add `@pytest.fixture(autouse=True)` (function-scoped, not session) that at the START of every test, re-reads `_GUI_DB_PATH` and re-anchors. This runs before monkeypatch.undo() because fixtures run before teardown:

```python
@pytest.fixture(autouse=True)
def _restore_gui_db():
    """Before each test: re-anchor DB_PATH to GUI test DB if gui_brain ran."""
    import engine.db as _db
    import engine.paths as _paths
    if _GUI_DB_PATH is not None:
        _db.DB_PATH = _GUI_DB_PATH
        _paths.DB_PATH = _GUI_DB_PATH
    yield
    # After test: re-anchor again (catches monkeypatch restores)
    if _GUI_DB_PATH is not None:
        _db.DB_PATH = _GUI_DB_PATH
        _paths.DB_PATH = _GUI_DB_PATH
```
