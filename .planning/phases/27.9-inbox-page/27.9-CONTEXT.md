# Phase 27.9: Inbox Page - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a dedicated Inbox tab showing a triage queue of: unassigned action items, recently-captured unprocessed notes, and empty notes. Users can navigate, dismiss, assign, delete, and link items without leaving the inbox. Creating new notes or action items is out of scope.

</domain>

<decisions>
## Implementation Decisions

### Sections & layout
- Three collapsible sections, displayed in this order: **Unassigned Actions → Unprocessed Notes → Empty Notes**
- All sections always visible even when count is 0 (consistent layout, no sections appearing/disappearing)
- Total count header at top of page: e.g. "12 items need attention"
- Unassigned Actions section supports **load more** (paginated) and **filter by source note** (the note the action came from)

### Split-view navigation
- Clicking a row opens the note/action in a **detail pane on the right** — list stays on the left
- No full navigation away from inbox; user can move between items in the list without losing context
- Similar to an email client (list + reading pane)

### Inline triage actions (per row, without opening full note)
- **Dismiss / mark as reviewed** — removes from inbox; persisted in SQLite (survives restarts); a `dismissed_inbox_items` table or flag on the note
- **Assign action item** — for Unassigned Actions section: pick an assignee directly from the row (reuse existing assignee picker pattern from ActionsPage)
- **Delete empty note** — for Empty Notes section: one-click delete with confirmation (reuse DeleteNoteModal or inline confirm)
- **Add backlink** — for Unprocessed Notes section: type-to-search picker to link to another note; adding a backlink marks the note as processed (removes from Unprocessed section)

### Definition of 'unprocessed'
- A note is unprocessed if: **created within the last 14 days** AND **has no tags** AND **has no backlinks/relationships**
- Only include freeform note types: `ideas`, `coding`, `strategy`, `personal`, `projects`
- Exclude: `people`, `meeting` (structured by design, don't need processing signal)

### Empty state / inbox zero
- When all item counts are 0: show "All clear — nothing to triage." with the timestamp it was last checked
- No celebration animation — clean and calm
- All three section headers still render even at 0 count (consistent layout)

### Claude's Discretion
- Exact dismiss persistence mechanism (DB flag vs separate table)
- Loading skeleton / loading state while fetching
- Exact styling of the split-view detail pane (reuse NoteViewer or a simplified variant)
- Backlink picker UX details (debounce, min-chars, max results)

</decisions>

<specifics>
## Specific Ideas

- "Open it in inbox page area so I can quickly navigate between different ones without going back and forth" — split-view is the key UX decision; user explicitly wants to stay in inbox while reviewing
- Actions section may have many items — implement load-more and source-note filtering to keep it manageable

</specifics>
