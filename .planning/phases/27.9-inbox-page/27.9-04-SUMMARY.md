# Phase 27.9 Plan 04: BacklinkPicker POST /search Fix Summary

## What Was Built

**Task 1:** Changed `BacklinkPicker` in `InboxPage.tsx` from:
```typescript
// BROKEN
fetch(`${getAPI()}/search?q=${encodeURIComponent(query)}`)
```
to:
```typescript
// CORRECT
fetch(`${getAPI()}/search`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ query }),
})
```
The `data.results` path was already correct. Rebuilt frontend bundle and reinstalled uv tool.

**Task 2:** Added `test_inbox_backlink_picker` to `tests/test_gui.py` — navigates to Inbox, opens BacklinkPicker by clicking "Add Backlink", types a query, and asserts:
1. Page does not contain "405" (would appear if GET was still used)
2. At least one result button appears in the picker
