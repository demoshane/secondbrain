## What Was Built

### InboxPage component (`frontend/src/components/InboxPage.tsx`)

Split-view layout:
- **Left pane (w-80):** header + three always-rendered collapsible sections
- **Right pane (flex-1):** detail pane or "Select an item to preview" empty state

Section 1 — Unassigned Actions:
- Source note filter input with 300ms debounce (passes `source_note` query param)
- Per-row inline assignee picker (shadcn Select, populated from GET /notes filtered to type person/people)
- Dismiss button calls POST /inbox/dismiss `{path, item_type: 'action'}`
- Load-more pagination via `actions_offset` param when `unassigned_actions_total > offset + 20`

Section 2 — Unprocessed Notes:
- BacklinkPicker sub-component: debounced GET /search (300ms, min 2 chars), dropdown of up to 10 results, POST /relationships on select
- Dismiss button
- Row click opens detail pane

Section 3 — Empty Notes:
- Delete button: `window.confirm` → DELETE /notes/<encoded_path>
- Dismiss button
- Row click opens detail pane

Header:
- `"N items need attention"` when total_count > 0
- `"All clear — nothing to triage."` + last-checked timestamp when total_count === 0

Detail pane:
- Fetches GET /notes/<path> on item selection
- Renders body via `window.marked` (already vendored)
- X button clears selection

### Tab wiring
- `'inbox'` added to View union in UIContext.tsx
- Inbox tab added to TABS array in TabBar.tsx (after Intelligence)
- `currentView === 'inbox'` branch added in App.tsx ternary (before final null)
- RightPanel condition unchanged: `{currentView === 'notes' && <RightPanel />}`

### Types (`frontend/src/types.ts`)
- `NoteSummary { path, title, created_at }`
- `InboxData { unassigned_actions, unassigned_actions_total, unprocessed_notes, empty_notes, total_count }`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] BacklinkPicker uses GET /search not POST /search**
- **Found during:** Task 2 implementation
- **Issue:** Plan specified `POST /search {query}` for backlink search, but the actual API endpoint is `GET /search?q=<query>` (confirmed from existing search usage in the codebase)
- **Fix:** Used `GET /search?q=encodeURIComponent(query)` in BacklinkPicker
- **Files modified:** frontend/src/components/InboxPage.tsx

**2. [Rule 1 - Bug] Removed unused `notePath` parameter from BacklinkPicker**
- **Found during:** Task 2 build (TypeScript TS6133 error)
- **Issue:** `notePath` was declared in BacklinkPicker props but never used inside the component (it's passed from parent at call site)
- **Fix:** Kept the prop in the type signature (for callers) but removed the destructured binding, then destructured only `onSelect` and `onCancel`
- **Commit:** 633d1a7
