# Phase 27.9 Plan 01: Inbox Backend API Summary

**One-liner:** Flask /inbox + /inbox/dismiss + /relationships endpoints with dismissed_inbox_items migration and full unit test coverage.

## What Was Built

Three Flask API endpoints backing the Inbox page, plus a DB migration for dismiss persistence.

### DB Migration (`engine/db.py`)

`migrate_add_dismissed_inbox_items_table()` creates:

```sql
CREATE TABLE IF NOT EXISTS dismissed_inbox_items (
    path         TEXT NOT NULL,
    item_type    TEXT NOT NULL,  -- 'note' | 'action'
    dismissed_at TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    PRIMARY KEY (path, item_type)
)
```

Called at the end of `init_schema()` so all existing DBs pick it up on startup.

### Endpoints (`engine/api.py`)

| Endpoint | Method | Behaviour |
|---|---|---|
| `/inbox` | GET | Returns unassigned_actions (paginated), unprocessed_notes (14-day window, no tags/rels, structured types), empty_notes; all filtered against dismissed_inbox_items |
| `/inbox/dismiss` | POST | INSERT OR IGNORE into dismissed_inbox_items; returns {dismissed: true} |
| `/relationships` | POST | INSERT OR IGNORE into relationships with rel_type='link'; returns {created: true} |

### Tests (`tests/test_inbox.py`)

5 unit tests covering:
1. `test_inbox_endpoint` — response shape (all 4 keys present)
2. `test_unprocessed_excludes_structured_types` — meeting/people notes excluded
3. `test_unprocessed_excludes_old_notes` — 15-day-old note excluded
4. `test_dismiss_persists` — dismissed empty note absent after POST /inbox/dismiss
5. `test_relationships_create` — row exists in relationships table after POST
