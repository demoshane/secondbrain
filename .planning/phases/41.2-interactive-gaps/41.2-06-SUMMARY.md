# Phase 41.2 Plan 06: Note Save Data Loss Fix Summary

**Critical data-loss bug fixed: NoteEditor now sends frontmatter-safe payloads via a new title+body combined API branch, preserving tags, type, people, and all other metadata on every save.**

## Accomplishments

- Added `title_and_body_title` + `title_and_body_body` combined branch to `save_note` in `engine/api.py` — reads frontmatter, updates both fields, writes atomically, updates DB
- Fixed `useNoteActions.ts` `saveNote` signature: accepts optional `title` param; sends combined payload when title is present, body-only payload when absent (never the destructive `{content}` path)
- Fixed `NoteEditor.tsx` `handleSave` to pass `title` state alongside `content` — title edits now persist

## Task Commits

1. **Task 1: Add title+body branch to save_note API** — `2eede3f` (feat)

## Files Created/Modified

- `engine/api.py` — new `title_and_body_*` branch inserted before full-content fallback in `save_note`
- `frontend/src/hooks/useNoteActions.ts` — `saveNote(path, body, title?)` with conditional payload
- `frontend/src/components/NoteEditor.tsx` — `handleSave` now calls `saveNote(note.path, content, title)`

## Decisions Made

- Used distinct key names `title_and_body_title` / `title_and_body_body` to sidestep collision with existing single-key branch guards (which explicitly check for absence of `title`, `body`, etc.)
- Frontend payload is conditional: title provided → combined branch; title absent → body-only branch. No caller ever hits the raw-content fallback from NoteEditor anymore.
