**One-liner:** Added Tags and People CollapsibleSections to ProjectsPage detail, fetching tags from `/notes/<path>` and people from existing `meta.people`.

## What Was Built

Extended `loadProjectDetail` to fetch a 4th endpoint (`GET /notes/${enc}`) in the existing `Promise.all`. The note data provides `tags: string[]`. The already-fetched `meta.people` array (`[{path, title}]`) was being discarded — it is now stored in `detailPeople` state.

Two new `CollapsibleSection` components render between the Linked Meetings and Related Notes sections:
- **Tags** — pills styled with `bg-secondary` and `#tag` prefix. Empty state: "No tags".
- **People** — pills styled with `bg-primary/10 text-primary`. Empty state: "No people linked".

Both sections default open when they have data, closed when empty.
