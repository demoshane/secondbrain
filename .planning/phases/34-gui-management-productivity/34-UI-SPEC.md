## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline button padding, tag chip gaps |
| sm | 8px | Compact element spacing, gap-2 between action buttons |
| md | 16px | Default element spacing, p-4 page padding (existing convention) |
| lg | 24px | Section padding, modal body padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used this phase) |

Exceptions:
- Cmd+K palette overlay: 8px internal item padding (compact list, not card layout)
- Tag autocomplete dropdown: max-h-40 (160px) to show ~5 suggestions before scroll
- Source note link icon button (ExternalLink): 32px touch target minimum (`h-8 w-8`, Tailwind class `size-8`)

## Color

All values are shadcn/ui CSS variable tokens (slate base, light and dark modes defined in index.css):

| Role | CSS Variable | Resolved (light) | Usage |
|------|-------------|------------------|-------|
| Dominant (60%) | `--background` | hsl(0 0% 100%) — white | Main surface, page background |
| Secondary (30%) | `--secondary` / `--muted` | hsl(210 40% 96.1%) — slate-100 | Cards, sidebar, table header, modal backdrop |
| Accent (10%) | `--primary` | hsl(222.2 47.4% 11.2%) — slate-900 | Primary action buttons only (see reserved list below) |
| Destructive | `--destructive` | hsl(0 84.2% 60.2%) — red-500 | Destructive actions only (delete entity, delete note) |

**Accent reserved for:**
- Primary CTA buttons: "New Person", "New Meeting", "New Project", "Create" (in entity create modals)
- Cmd+K palette active/selected item highlight
- Action item checkbox checked state

**Accent is NOT used for:** navigation links, tag badges, filter selects, source-note icon, table rows, toast backgrounds.

**Muted foreground** (`--muted-foreground`, hsl(215.4 16.3% 46.9%)): secondary text — due dates, empty state body, metadata labels. Use `text-muted-foreground` class consistently.

**Destructive variant** is already in the existing shadcn theme and applied via `variant="destructive"` on Button — no new tokens needed.

## Interaction Contracts

### ActionItemList (shared component)

- Checkbox: Radix Checkbox (existing `ui/checkbox.tsx`), `checked` = done state, `onCheckedChange` calls toggleDone
- Assignee picker: Radix Select (existing `ui/select.tsx`), h-7 w-36 text-xs (matches ActionsPage pattern)
- Source note link: `<Button variant="ghost" size="icon" className="h-8 w-8">` wrapping `<ExternalLink size={14} />` — visible only when `note_path` is present
- Done items: `line-through text-muted-foreground` on task text — visually distinct but not removed from list
- Empty state: render inline message, not a full-page empty state (it is embedded in larger surfaces)

### CommandPalette (Cmd+K)

- Trigger: `document.addEventListener('keydown')` in App.tsx, match `(e.metaKey || e.ctrlKey) && e.key === 'k'`
- Overlay: fixed full-screen backdrop (bg-black/50), centered modal, max-w-lg, using cmdk `<Command>` primitive
- Input: cmdk `<CommandInput>` with placeholder "Type a command or search…"
- Groups: "Navigation" (jump to note by title, switch pages) and "Capture" (quick-capture, open SmartCaptureModal)
- Dismiss: Escape key, click-outside — standard cmdk behavior
- Keyboard: arrow keys navigate, Enter selects — cmdk handles natively

### NewEntityModal

- Follows NewNoteModal.tsx contract exactly:
  - `<Dialog>` + `<DialogContent>` + `<DialogHeader><DialogTitle>`
  - Required field: `<Input placeholder="Name" />` — Enter key submits
  - People modal only: optional `<Input placeholder="Role / title (optional)" />`
  - Actions: `<Button variant="outline">Discard</Button>` + `<Button disabled={!name.trim() || creating}>Create</Button>`
  - Loading state: button text changes to "Creating…" while in-flight
- Modal title: "New Person" / "New Meeting" / "New Project" (specific, not generic "New Entity")

### DeleteEntityModal

- Extends DeleteNoteModal.tsx:
  - Same Dialog structure, same button pair (`variant="outline"` dismiss + `variant="destructive"` Delete)
  - Dismiss button label is entity-specific: "Keep Person" / "Keep Meeting" / "Keep Project"
  - Adds a "Linked data" section between title and button row:
    - `<p className="text-sm text-muted-foreground">` showing counts: "X meeting notes mention this person. Y action items are assigned to them."
    - If counts are zero: section is omitted
  - Loading state: "Deleting…" on destructive button (matches DeleteNoteModal pattern)

### TagAutocomplete

- Wraps existing tag `<Input>` in NoteViewer; on first keystroke, fetches `GET /tags`, then filters client-side
- Dropdown: absolute-positioned `<ul>` below input, `bg-popover border border-border rounded-md shadow-md`
- Max visible: 5 suggestions (max-h-[160px] overflow-auto)
- Keyboard: ArrowDown/ArrowUp navigate highlighted item, Enter selects, Escape dismisses
- Click-outside dismisses via `useEffect` with `mousedown` listener
- Selected suggestion replaces current tag input value and closes dropdown

### Toast (sonner)

- Mount `<Toaster />` once in App.tsx
- Position: bottom-right
- Duration: 3000ms (default)
- Triggers (all use `toast.success` or `toast.error`):
  - Action item toggled done: `toast.success("Marked complete")`
  - Action item toggled undone: `toast.success("Marked open")`
  - Entity created: `toast.success("[Name] created")`
  - Entity deleted: `toast.success("[Name] deleted")`
  - Tag saved: `toast.success("Tags saved")`
  - Any mutation failure: `toast.error("Something went wrong — try again")`

| Action | Confirmation Approach |
|--------|-----------------------|
