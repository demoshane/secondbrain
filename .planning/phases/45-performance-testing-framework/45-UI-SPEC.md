## Color

All values are CSS custom properties defined in `frontend/src/index.css`. Use the CSS variable references below — do not hardcode hex.

| Role | CSS Variable | Resolved Hex | Usage |
|------|-------------|--------------|-------|
| Dominant (60%) | `hsl(var(--background))` | #0f1117 | Page background, main content area |
| Secondary (30%) | `hsl(var(--card))` | #151b27 | Summary table container, sparkline card backgrounds |
| Accent (10%) | `hsl(var(--primary))` | #5b6cf7 | Gauge tab icon active state, SVG sparkline stroke color |
| Destructive | `hsl(var(--destructive))` | #ef4444 | `warn` / `error` status rows — regression beyond limit (red delta) |

Accent reserved for:
- Performance tab icon when active (TabBar underline/highlight)
- SVG sparkline polyline stroke
- Focus rings on any interactive element (`ring` token, same as `--primary`)

Semantic status colors (not in the 60/30/10 split — used only in status column):
- Pass: `text-green-400` — elapsed within limit
- Warn: `text-amber-400` with `⚠` prefix — elapsed beyond limit
- Error: `text-red-400` — tool threw exception

Delta display colors (D-19 from CONTEXT.md):
- Regression beyond limit: `text-red-400`, prefix `+`
- Improvement: `text-green-400`, prefix `-`
- Within-limit change (either direction): `text-muted-foreground` (grey), prefix `+` or `-`

Source: `frontend/src/index.css` CSS variable block, D-19 decision in CONTEXT.md, existing status color patterns from IntelligencePage.tsx urgency badges.

## Page Layout: PerformancePage

Layout follows IntelligencePage.tsx conventions: full-width scrollable page, no sidebar interaction, CollapsibleSection wrappers for grouping.

```
PerformancePage
├── Page header (h1 "Performance" + muted subtitle)
├── CollapsibleSection "Latest Run"
│   └── Summary table (Tool | Latest | Previous | Delta | Limit | Status)
│       — sticky header row
│       — one row per tool_result from GET /perf/results/latest
│       — "No results yet" empty state when API returns null latest
└── CollapsibleSection "30-Day Trend"
    └── Grid of sparkline cards (2-col on wide, 1-col on narrow)
        — each card: tool name + Sparkline SVG
        — sparkline data from GET /perf/results/<date> for each available date
```

The page is read-only. No "Run Benchmark" button. No destructive actions.

## States to Implement

| State | Trigger | UI Response |
|-------|---------|-------------|
| Loading | On mount, before API responds | SkeletonList in table area (existing component) |
| Loaded with data | API returns latest result | Render summary table + sparklines |
| Loaded, no results | API returns null latest | EmptyState component (existing) with copy above |
| API error | fetch() throws or non-200 | Error copy in muted text, no crash |
| Single result (no delta) | Only one result file exists | Previous = "—", Delta = "—" |
| Tool error row | tool_result.status === 'error' | Red `error` badge, elapsed shown, error message as tooltip if available |

Source: D-21 (previous=null if one result), RESEARCH.md Pitfall 4 (API down scenario).
