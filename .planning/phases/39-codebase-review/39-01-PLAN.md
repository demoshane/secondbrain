    Systematically audit every public surface for security issues. Per D-04, focus on input validation, path traversal, SQL injection, PII exposure.

    **Backend API (engine/api.py):**
    - Every `request.args.get()` and `request.json` access: is it validated? Type-checked? Bounds-checked?
    - Known pre-identified: S-01 (unguarded int() on lines ~165, 321, 337, 406, 640, 738, 1108, 1456), S-02 (duplicate import lines 24-25), S-06 (file delete path validation)
    - Path parameters: do they all go through `safe_join()` or `is_relative_to()` checks?
    - SQL queries: any string interpolation (f-strings or .format()) going into SQL?
    - Error responses: do they leak internal paths or stack traces?

    **MCP server (engine/mcp_server.py):**
    - Every tool function: are arguments validated before use?
    - Destructive tools: do they all use two-step confirm_token pattern?
    - PII: does any tool return raw note content without sensitivity filtering?

    **Chrome extension:**
    - content.js: what data is read from pages? Scope of `<all_urls>`?
    - popup.js: any `innerHTML` usage? Is `escapeHtml()` consistently applied?
    - CSP in manifest: adequate?
    - Message passing: validated?

    **Other write-path modules:**
    - capture.py, forget.py, delete.py, anonymize.py, backup.py: input validation on public functions

    Write findings to `39-findings-security.md` using format:
    ```
    # Security Audit Findings

    ## Finding: SEC-NN
    - **Severity:** Critical/High/Medium/Low
    - **File:** path:line
    - **Description:** what's wrong
    - **Root cause:** why
    - **Recommended fix:** specific action
    - **Blast radius:** what breaks if fixed wrong
    ```

    Severity calibration per D-05:
    - Critical = data loss or security breach risk
    - High = broken behavior, significant correctness issue
    - Medium = meaningful quality issue
    - Low = cosmetic, preference, minor polish
