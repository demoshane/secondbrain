  <action>
SQLite does not support `ALTER TABLE ... ADD CONSTRAINT`. The established migration pattern in this codebase is to recreate tables with the correct schema inside `init_schema()`.

1. **Add a migration function `_migrate_fk_cascade(conn)`** that:
   a. Checks if migration is needed (e.g., check if note_embeddings FK already has CASCADE — use `PRAGMA foreign_key_list(note_embeddings)` and look for `on_delete='CASCADE'`; if already present, skip)
   b. For each of the 3 tables, use the rename-recreate pattern:
      ```python
      # note_embeddings
      conn.execute("ALTER TABLE note_embeddings RENAME TO _ne_old")
      conn.execute("""CREATE TABLE note_embeddings (
          note_path    TEXT PRIMARY KEY REFERENCES notes(path) ON DELETE CASCADE,
          embedding    BLOB,
          content_hash TEXT,
          stale        BOOL NOT NULL DEFAULT 0,
          created_at   TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
          updated_at   TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
      )""")
      conn.execute("INSERT INTO note_embeddings SELECT * FROM _ne_old")
      conn.execute("DROP TABLE _ne_old")
      ```
   c. Same pattern for `action_items` — add `REFERENCES notes(path) ON DELETE CASCADE` to `note_path`:
      ```python
      conn.execute("ALTER TABLE action_items RENAME TO _ai_old")
      conn.execute("""CREATE TABLE action_items (
          id         INTEGER PRIMARY KEY,
          note_path  TEXT NOT NULL REFERENCES notes(path) ON DELETE CASCADE,
          text       TEXT NOT NULL,
          done       BOOL NOT NULL DEFAULT 0,
          created_at TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
      )""")
      conn.execute("INSERT INTO action_items SELECT * FROM _ai_old")
      conn.execute("DROP TABLE _ai_old")
      ```
      NOTE: action_items may have additional columns added by prior migrations (e.g., `assignee_path`, `due_date`). READ the actual schema first via `PRAGMA table_info(action_items)` and include ALL columns in the recreated table.
   d. Same pattern for `relationships` — add CASCADE on BOTH `source_path` and `target_path`:
      ```python
      # Check PRAGMA table_info(relationships) for actual columns first
      conn.execute("ALTER TABLE relationships RENAME TO _rel_old")
      conn.execute("""CREATE TABLE relationships (
          id          INTEGER PRIMARY KEY,
          source_path TEXT NOT NULL REFERENCES notes(path) ON DELETE CASCADE,
          target_path TEXT NOT NULL REFERENCES notes(path) ON DELETE CASCADE,
          rel_type    TEXT NOT NULL,
          created_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
      )""")
      conn.execute("INSERT INTO relationships SELECT * FROM _rel_old")
      conn.execute("DROP TABLE _rel_old")
      ```
      Again, check actual schema first and include all columns.
   e. Do NOT add CASCADE to `audit_log` — audit entries must survive note deletion for compliance.

2. **Add indexes** (idempotent, use IF NOT EXISTS):
   ```python
   conn.execute("CREATE INDEX IF NOT EXISTS idx_notes_archived ON notes(archived)")
   conn.execute("CREATE INDEX IF NOT EXISTS idx_action_items_note_path ON action_items(note_path)")
   ```

3. **Wire into init_schema()**: Call `_migrate_fk_cascade(conn)` and the index statements from `init_schema()`, after existing migrations. Wrap in try/except to handle already-migrated DBs gracefully.

IMPORTANT: Before recreating any table, run `PRAGMA table_info(table_name)` and use the ACTUAL column list. Do not assume schema from the static SCHEMA_SQL — migrations may have added columns.
