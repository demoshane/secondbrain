<objective>
Add a "Summarise page" feature to the Chrome extension popup. The user clicks one button, the
active tab's content is extracted and sent to the backend, the LLM produces a summary, and the
user can save it to the brain with a single click. Zero clipboard fiddling.
</objective>

<context>
@chrome-extension/popup.html
@chrome-extension/popup.js
@engine/api.py (lines 1-50 for imports/setup)
</context>

<tasks>

<task type="auto" tdd="true">
<behavior>Backend: POST /summarise-url endpoint</behavior>
<action>
Add POST /summarise-url to engine/api.py.
Request body: {url: str, content: str (extracted page text, max 8000 chars)}.
Calls LLM via _router.get_adapter('public') to produce a 1–3 paragraph summary.
Returns {summary: str}.
Guard input: abort(400) if url or content missing/empty; truncate content to 8000 chars server-side.
</action>
<verify><automated>uv run pytest tests/ -k summarise -x -q</automated></verify>
<done>POST /summarise-url returns {summary} for valid input; 400 for missing fields.</done>
</task>

<task type="auto" tdd="false">
<behavior>Extension: summarise button + flow</behavior>
<action>
In popup.html: add a "Summarise page" button below the main capture form.
In popup.js:
- On click: show spinner, extract document text via chrome.tabs.executeScript (innerText, max 8000 chars), POST to /summarise-url.
- On success: hide form, show summary panel with the text and an "Add to brain" button.
- "Add to brain": POST /notes with type=note, title="Summary: {page title}", body=summary, source_url=tab.url.
- On error: show inline error message.
</action>
<verify><automated>echo "Manual: load extension, open any page, click Summarise, verify spinner → summary → save"</automated></verify>
<done>Full summarise → save flow works end-to-end in Chrome.</done>
</task>

</tasks>
