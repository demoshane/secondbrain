# Phase 41.3 Plan 07: Chrome Extension — Page Summarisation Summary

One-liner: Single-click page summarisation in Chrome extension popup — LLM-generated 1-3 paragraph summary with inline display and one-click save to brain.

## What Was Built

**Task 1: POST /summarise-url backend endpoint (TDD)**

New Flask endpoint in `engine/api.py` that:
- Accepts `{url: str, content: str}` JSON
- Guards: 400 if url or content missing/empty
- Truncates content to 8000 chars server-side
- Calls `_router.get_adapter('public')` with a concise summarisation system prompt
- Returns `{summary: str}`; 500 on LLM exception

Six tests in `tests/test_summarise_url.py`: 400 guards for missing/empty fields, happy path with mocked adapter, truncation verification.

**Task 2: Extension summarise button + flow**

`chrome-extension/popup.html` — added `#summarise-section` with a full-width "Summarise page" button and a hidden `#summary-panel` with summary text display, "Add to brain" button, and "Discard" button.

`chrome-extension/popup.js` — three new functions:
- `handleSummarise()`: shows spinner, extracts page text via existing `sendToContentScript` → `extract-article`, POSTs to `/summarise-url`, shows summary panel on success, inline errors on failure
- `handleAddTobrainFromSummary()`: POSTs to `/notes` with `type=note`, `tags=['summary']`, `source_url`, closes popup on success
- `discardSummary()`: resets to capture form

All three handlers attached unconditionally before the Gmail URL-param early return path.

`chrome-extension/popup.css` — styles for `.summarise-section`, `#summarise-btn`, `#summary-panel`, `.summary-text`, `#add-to-brain-btn`, `#summary-cancel-btn`.
