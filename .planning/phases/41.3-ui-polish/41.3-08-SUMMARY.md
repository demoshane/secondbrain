# Phase 41.3 Plan 08: NoteViewer — [[wiki-links]] clickable navigation Summary

**One-liner:** [[wiki-link]] syntax in NoteViewer body renders as clickable chips that navigate to the linked note by case-insensitive title match, with dimmed fallback for unresolved links.

## What Was Built

Added Obsidian-style `[[wiki-link]]` navigation to NoteViewer. Clicking a `[[Note Title]]` in a note body switches view to the Notes tab and opens the matched note directly. Unresolved links (no note with that title) appear as dimmed non-interactive chips.

## Implementation

- `preprocessWikiLinks(body)`: pure function that regex-replaces `[[Title]]` → `[Title](wiki:Title%20encoded)` in the body string before passing to ReactMarkdown
- `markdownComponents.a`: custom ReactMarkdown anchor renderer that intercepts `wiki:` scheme hrefs, decodes the title, looks it up in the `titleToPath` map, and either renders a navigating `<button>` (matched) or a dimmed `<span>` (unmatched)
- Regular `http/https` links fall through to a standard `<a target="_blank">` render unchanged
- `titleToPath` is a `useCallback` that builds a case-insensitive `Map<string, path>` from the `notes` array — recomputed only when notes list changes
