# Phase 41.3 Plan 06: Attachments Section on All Detail Panels Summary

**One-liner:** Reusable `AttachmentsSection` component with live refresh via `refreshTick` wired to all five detail panels (NoteViewer, LinksPage, PeoplePage, MeetingsPage, ProjectsPage).

## What Was Built

**AttachmentsSection component** (`frontend/src/components/ui/attachments-section.tsx`):
- Props: `notePath: string`, `refreshTick?: number`, `onUploadClick?: () => void`
- Fetches `GET /notes/attachments?path={notePath}` on mount and whenever `refreshTick` changes
- Always visible — shows "No attachments yet" when empty
- Renders Paperclip header + file list as anchor links + optional Upload button

**FileUploadModal improvements:**
- `onUploaded?: () => void` — called after successful upload, before `onClose()` — allows callers to trigger refresh
- `notePath?: string` prop — overrides `NoteContext.currentPath`; required for LinksPage, PeoplePage, MeetingsPage, ProjectsPage where the contextual currentPath is not the selected entity

**NoteViewer:**
- Replaced inline attachment block (lines 159-176) with `<AttachmentsSection>`
- Added `refreshTick` state (starts 0, resets to 0 on note.path change, incremented on upload via `onUploaded`)
- Fixes the original bug: upload no longer requires navigation to see new file

**LinksPage, PeoplePage, MeetingsPage, ProjectsPage:**
- Each adds `AttachmentsSection` + `FileUploadModal` to their detail panel
- State: `showUpload` (boolean), `attachRefreshTick` (number, resets to 0 on entity selection change)
- `FileUploadModal` passes `notePath={selectedPath}` to bypass NoteContext

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical Functionality] FileUploadModal.notePath prop added**
- **Found during:** Task 1, implementation of non-NoteViewer pages
- **Issue:** FileUploadModal uses `currentPath` from `NoteContext` — on LinksPage, PeoplePage, MeetingsPage, ProjectsPage the selected entity path is not reflected in NoteContext's `currentPath`, causing uploads to attach to the wrong note or fail silently
- **Fix:** Added optional `notePath` prop to `FileUploadModal` that overrides `currentPath` when provided; all entity pages pass their `selectedPath`
- **Files modified:** `frontend/src/components/FileUploadModal.tsx`
- **Commit:** 14fe111
