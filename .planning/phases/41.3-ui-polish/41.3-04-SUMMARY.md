# Phase 41.3 Plan 04: Inline note editing on detail pages — Summary

Inline body editing added to all four detail-panel pages (Links, People, Meetings, Projects). Users can now edit the selected note's content in-place without navigating away.

## What Was Built

Each detail panel now has:
- A pencil **Edit** button in the header toolbar, visible only in read mode
- Clicking Edit switches the body area to a full-height resizable monospace `<textarea>` pre-filled with the raw note body
- **Save** button (or Cmd+Enter) calls `PUT /notes/{encodedPath}` with `{ body: editingBody }`, updates local state, and returns to read view
- **Cancel** (or Escape) discards changes and returns to read view
- **Open as Note** is unchanged on all pages

Page-specific notes:
- **LinksPage**: body toggle replaces the existing `{linkDetail?.body && ...}` block
- **PeoplePage**: body toggle replaces the body inside the `CollapsibleSection` "Profile & Context"; Edit button placed alongside "Open in Notes"
- **MeetingsPage**: body toggle replaces the body inside the `CollapsibleSection` "Notes"; Edit button placed alongside "Open in Notes"
- **ProjectsPage**: body rendering was entirely absent — added a new "Notes" `CollapsibleSection` that shows/edits the body, plus the existing "Related Notes" section (backlinks list) is kept separate. Edit button alongside "Open in Notes".

## API Contract

`PUT /notes/<path>` with `{ body: "..." }` — this hits the body-only branch in `engine/api.py` which preserves frontmatter and writes only the content portion. No new backend changes needed.
