## What Was Built

**ActionItemRow changes:**
- New `onSetDue?: (id: number, date: string | null) => void` prop
- `editingDue` state drives date input visibility
- When due date exists: shows date text with pencil icon on hover; click to enter edit mode
- When no due date and `onSetDue` provided: shows "Set deadline" ghost button on row hover
- Edit mode: `<input type="date">` pre-filled with current value, ✓ confirm and ✕ cancel buttons
- Confirm with empty value passes `null` to clear the deadline
- Overdue red styling unchanged when not editing

**PeoplePage + ProjectsPage (identical pattern):**
- `handleSetDue`: PUTs `{ due_date }` to `/actions/<id>`, then reloadActions()
- `onSetDue={handleSetDue}` passed to each ActionItemRow in Actions section
- `showAddAction` + `newActionText` state for inline toggle form
- `handleAddAction`: POSTs `{ text, note_path: selectedPath }` to `/actions`, then reloadActions()
- "+ Add action" text button toggles to inline input with Add/Cancel controls
- Enter key submits, Escape cancels
