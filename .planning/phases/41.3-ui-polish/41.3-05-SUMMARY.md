# Phase 41.3 Plan 05: Connections panel — proper sections, tags, add/remove — Summary

**One-liner:** RightPanel now shows a filterable Tags section and People section with inline add/remove controls backed by frontmatter-safe PUT endpoints.

## What Was Built

### Task 1 — API changes (engine/api.py)
- `note_meta()`: extended SELECT to include `tags` column alongside `people, body`
- Added `note_tags = json.loads(note_row["tags"] or "[]")` extraction
- Removed the `else` branch that appended `{"path": None, "title": item_str}` for plain-text names that didn't resolve to a real person note — these are noise from old entity extraction bugs
- Response now includes `"tags": [...]` array

The `people` branch in `save_note()` already existed from a previous plan — no changes needed there.

### Tasks 2 + 3 — RightPanel.tsx
- Added `tags`, `tagInput`, `personQuery`, `personResults`, `showPersonDropdown` state
- Extracted `reloadMeta` as a `useCallback` so it can be called after tag/person mutations
- `addTag(tag)` / `removeTag(tag)`: PUT to `/notes/{path}` with `{tags: updatedList}`, then reload meta
- `addPerson(path)` / `removePerson(path)`: PUT to `/notes/{path}` with `{people: updatedPaths}`, then reload meta
- `handlePersonQueryChange`: debounced (250ms) POST to `/search` with `{query, note_type: "person", limit: 5}` — populates dropdown
- Tags section: always rendered when `currentPath` is set; TagBadge chips with × remove; inline `+ tag` input (Enter or blur commits)
- People section: always rendered when `currentPath` is set; PersonBadge chips with × remove; inline `+ person` typeahead with dropdown; uses `onMouseDown` to beat the blur event
- Collapsed panel: Tag icon shown when `tags.length > 0`

## Deviations from Plan

### Deviation: People branch already existed
- **Found during:** Task 3 pre-implementation read
- **Issue:** Plan says "New PUT /notes/{path} people branch needed in api.py" but it was already implemented in a prior plan (lines 1151–1179 of api.py)
- **Action:** Skipped API change for people; only frontend changes needed
- This is not a bug or deviation that required a fix — the plan was based on a slightly stale assessment

## Known Stubs

None. Tags and People sections fetch live data from the API on every `currentPath` change.
