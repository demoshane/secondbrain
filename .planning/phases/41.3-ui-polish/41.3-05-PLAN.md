<objective>
The Connections panel currently mixes real person notes with unresolved plain-text strings
("Change Management", "Maturity Label", "Open Items") from old bad entity extraction — all
shown as person chips with no way to distinguish them.  Tags are invisible: notes have them
in the DB but the panel never shows them and there's no way to add or remove tags or
connections from any note view.

This plan fixes the People section, adds a Tags section, and adds inline add/remove controls
for both — on RightPanel and everywhere a note is viewed.
</objective>

<context>
  @engine/api.py          (note_meta endpoint, PUT /notes/{path} tags branch)
  @frontend/src/components/RightPanel.tsx
  @frontend/src/components/ui/tag-badge.tsx
  @frontend/src/components/ui/person-badge.tsx
</context>

<tasks>

<task type="auto" tdd="false">
  <name>API: filter unresolved people from meta endpoint</name>
  <behavior>
    The `/notes/{path}/meta` endpoint appends `{"path": None, "title": item_str}` for any
    people entry that doesn't match a real person note.  These are noise from old entity
    extraction bugs.  Remove them: only include people items where a matching person-type
    note was found in the DB (path is not None).  Also add `tags` field to the response:
    read from `notes.tags` JSON column for the current note.
  </behavior>
  <action>
    In `note_meta()` in api.py:
    1. Remove the `else` branch that appends `{"path": None, "title": item_str}` (lines ~1350–1354).
    2. Read `notes.tags` alongside `notes.people` in the existing SELECT.
    3. Add `"tags": json.loads(note_row["tags"] or "[]")` to the jsonify response dict.
  </action>
  <verify>
    <automated>uv run pytest tests/test_api.py -q -k "meta" 2>&1 | tail -5</automated>
  </verify>
  <done>GET /notes/{path}/meta returns only people with real person-note paths; response includes `tags` array.</done>
</task>

<task type="auto" tdd="false">
  <name>RightPanel: Tags section + add/remove controls</name>
  <behavior>
    - Fetch `tags` from the meta response.
    - Show a "Tags" CollapsibleSection below Backlinks and above People, even when empty
      (so there's always a place to add tags).
    - Each tag renders as a TagBadge chip with an × remove button.
    - Below the chips: a small inline input ("+ tag") that on Enter/blur saves the updated
      tag list via `PUT /notes/{path}` with `{"tags": [...]}`.
    - Removing a tag also fires the same PUT.
    - Reload tags from meta after each save.
  </behavior>
  <action>
    In RightPanel.tsx:
    1. Add `tags` state, populated from `/meta` response alongside `people`.
    2. Add `addTag(tag: string)` and `removeTag(tag: string)` helpers that call
       `PUT /notes/{encoded}` with `{tags: updatedList}` then refetch meta.
    3. Render a Tags CollapsibleSection: existing TagBadge chips each with × icon,
       followed by an inline `<input placeholder="+ tag" />` that commits on Enter/blur.
    4. Show Tags section always (not just when tags.length > 0) so it's discoverable.
  </action>
  <verify>
    <automated>uv run pytest tests/ -q -k "tag" 2>&1 | tail -5</automated>
  </verify>
