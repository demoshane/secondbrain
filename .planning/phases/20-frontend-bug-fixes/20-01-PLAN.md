<objective>
Fix the two coupled API bugs: frontmatter stripping (GUIX-03) and save + SQLite re-index (GUIX-02).

These must be fixed together because stripping frontmatter from the viewer response means the editor can no longer fetch the full file via the same endpoint — if the save then writes body-only content, notes permanently lose their `title:`, `tags:`, and `type:` YAML fields. The plan adds a `?raw=true` query parameter so the editor fetches raw (full) content while the viewer gets clean body-only HTML.

Purpose: Notes render as formatted HTML, title edits persist to SQLite and appear in the sidebar.
Output: Updated `engine/api.py` with frontmatter strip + raw param + post-save re-index; updated `tests/test_api.py` with TestReadNote extensions and new TestSaveNote class.
</objective>
