<objective>
Fix the CSS scroll bug in the note viewer (GUIX-04).

The root cause: `#center { overflow: hidden }` combined with flex layout prevents `#viewer { overflow-y: auto }` from creating a scrollable region. The canonical fix is `min-height: 0` on the flex container (`#center`) so flex children can shrink and trigger overflow-scroll.

Purpose: User can scroll long notes with the mouse wheel.
Output: 2-line CSS change in `style.css`; human verification that scroll and editor layout both work.
</objective>
