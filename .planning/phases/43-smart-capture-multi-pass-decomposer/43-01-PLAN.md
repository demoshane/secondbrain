    **In `engine/typeclassifier.py` (line 81):**
    Remove the URL hard-override entirely. Delete these two lines:
    ```python
    # link — URL presence is deterministic, always wins
    if _URL_PAT.search(combined):
        return ("link", 1.0)
    ```

    Instead, add a "link" scoring section alongside the other types (meeting, person, coding, etc.) in the `scores` dict. The link score should be:
    - 0.70 if body contains 1+ URLs (weak signal — URL present but content may be about something else)
    - 0.85 if body is predominantly a URL (body stripped of URLs is < 50 chars — the URL IS the content)
    - This allows meeting notes with Zoom URLs to still classify as "meeting" when meeting keywords score higher than 0.70

    Add link scoring after the existing `scores` dict assignments (before the `pick best` section):
    ```python
    # ------------------------------------------------------------------ link
    url_count = len(_URL_PAT.findall(combined))
    url_stripped = _URL_PAT.sub('', combined).strip()
    s = 0.0
    if url_count > 0:
        if len(url_stripped) < 50:
            s = 0.85  # URL is the primary content
        else:
            s = 0.70  # URL present but content has substance
    scores["link"] = s
    ```
