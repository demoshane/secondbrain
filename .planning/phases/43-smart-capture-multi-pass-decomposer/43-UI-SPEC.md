## Spacing Scale

Declared values (multiples of 4 — standard 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), inline padding |
| sm | 8px | Compact row padding, input inner padding (`px-2 py-1`) |
| md | 16px | Default element spacing, textarea horizontal padding |
| lg | 24px | Modal section gaps (`space-y-6`) |
| xl | 32px | Not used in modal context |
| 2xl | 48px | Not used in modal context |
| 3xl | 64px | Not used in modal context |

Exceptions:
- Marker tag chips: 6px vertical padding (`py-1.5`) — sits between sm and md, per existing badge pattern
- Action-item marker input height: 32px (`h-8`) — matches existing Input/Select height in SettingsModal
- Scrollable list max-height: 160px (`max-h-40`) for saved notes list; 192px (`max-h-48`) for pending review list — matches SmartCaptureModal pattern

## Component Inventory

### SmartCaptureModal — result section additions (Phase 43 delta)

The existing `SmartCaptureModal.tsx` Step 2 result view gains two new display areas:

**1. Person stubs created row**
When `result.person_stubs` is non-empty, display below the saved notes list:
- Layout: same `space-y-1.5` list pattern as saved notes
- Each row: `<User className="h-4 w-4 text-muted-foreground shrink-0" />` + badge `person` (using existing `TYPE_COLORS.person`) + stub name truncated
- Section label: `"N person stub{s} created"` in `text-sm text-muted-foreground`
- No interaction — display only (stubs are already saved)

**2. Link notes extracted row**
When `result.link_notes` is non-empty, display as a sub-section within the saved notes list — link notes appear as regular saved entries with the `link` badge, no separate section needed. They flow into the existing `saved` array returned from the API.

### SettingsModal — new "Capture" section

Add a third section to `SettingsModal.tsx` below the existing "Ollama" section.

**Section header:** `"Capture"` — matches existing section header pattern (`text-sm font-medium mb-3`)

**Action-item markers subsection:**
- Label: `"Action-item markers"` (`text-xs font-medium text-foreground`)
- Description: `"Keywords that flag a line as an action item during smart capture"` (`text-xs text-muted-foreground mb-1.5`)
- Marker list: horizontal flex-wrap of tag chips
  - Each chip: `inline-flex items-center gap-1 px-2 py-1 rounded text-xs bg-secondary border border-border`
  - Remove button inside chip: `<X className="h-3 w-3" />` with `hover:text-destructive` transition; aria-label `"Remove marker {name}"`
  - Default markers (`TODO`, `AP`, `action:`, `@owner`, `Action Point`) displayed as non-removable chips (no X button, slightly different visual: `text-muted-foreground` text color instead of `text-foreground`)
- Add new marker row: `<Input h-8 placeholder="Add marker..." />` + `<Button size="sm">Add</Button>` in a flex row
  - Input accepts text; pressing Enter or clicking Add triggers the add action
  - Empty or whitespace-only input: Add button disabled
  - Duplicate marker: Add button disabled + input border turns destructive (no toast, inline validation only)
- Save interaction: markers are saved along with the rest of the config via the existing Save button at the bottom of SettingsModal. No separate save for this section.

**Shadcn components used in new section:** `Input`, `Button` — already installed in the project.

## Interaction States

### SmartCaptureModal (person stub display)

| State | Behaviour |
|-------|-----------|
| 0 stubs, 0 link notes | No change — result view identical to pre-Phase-43 |
| N stubs created | New sub-section appears below saved notes list; display only |
| N link notes created | Link notes appear as `link`-type rows in the main saved notes list |
| Mix of stubs + link notes + primary notes | All three appear; stub section is last, after saved notes |

### SettingsModal — markers panel

| State | Behaviour |
|-------|-----------|
| Initial load | Markers fetched alongside routing/ollama config via `GET /config`; display immediately |
| Loading | Existing `Loader2` spinner covers full modal — no per-section loading state |
| Add: valid new marker | Chip appears in list; input clears; Save button remains enabled |
| Add: duplicate | Input border `border-destructive`; Add button disabled; no chip added |
| Add: Enter key | Same as clicking Add button |
| Remove custom marker | Chip removed from list immediately; Save button remains enabled |
| Remove default marker | Not possible — no X button rendered |
| Save | Full config written (routing + ollama + action_items.custom_markers); "Saved" confirmation shown |
