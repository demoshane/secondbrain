    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      test:
        runs-on: ubuntu-latest

        steps:
          - uses: actions/checkout@v4

          - name: Set up uv
            uses: astral-sh/setup-uv@v4
            with:
              enable-cache: true

          - name: Install dependencies
            run: uv sync --dev

          - name: Run tests
            run: uv run pytest tests/ -q --tb=short
            env:
              BRAIN_PATH: /tmp/test-brain
    ```

    Notes:
    - Python version: do NOT pin python-version in the workflow — uv reads .python-version from repo
    - BRAIN_PATH env var set to /tmp/test-brain: tests that don't monkeypatch this will use a clean temp dir
    - enable-cache: true for faster runs (uv cache)
    - astral-sh/setup-uv@v4 is the standard action for uv (stable, widely used)
    - ubuntu-latest is sufficient — this project has no macOS-specific test requirements (pywebview GUI tests require macOS, but those are excluded from the suite or already gated)
    - If any existing tests import pywebview or require macOS, those tests should already be skipped on non-macOS (check existing test files for platform guards; if absent, add pytest.mark.skipif for pywebview-dependent tests as needed)
