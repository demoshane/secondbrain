# Phase 27.4 Plan 01: People Page — Wave 0 Test Scaffolds Summary

**One-liner:** xfail unit tests for /api/people endpoint + Playwright e2e stubs + PersonSummary TypeScript type contract establishing Wave 0 RED baseline for the People page feature.

## What Was Built

Wave 0 scaffolds for the People page feature. No implementation — only the test contracts that Wave 1 (backend endpoint) and Wave 2 (frontend component) will make green.

### tests/test_people.py (new file)
Three xfail unit tests using the Flask test client pattern:

- `test_list_people_endpoint` — GET /people returns 200 with a `people` key
- `test_list_people_empty` — no people notes returns `{"people": []}`
- `test_list_people_stats` — person note in `people/` folder + 1 open action item → `open_actions=1`, `updated_at` present

The `client` fixture isolates each test with a tmp brain dir, patches `engine.db.DB_PATH` and `engine.paths.DB_PATH`, and sets `BRAIN_PATH` env var — matching the established pattern from Phase 24/25.

### tests/test_gui.py (3 stubs added)
Three xfail Playwright stubs appended adjacent to existing GUI tests:

- `test_people_tab_visible` — People button visible in `[data-testid='tab-bar']`
- `test_people_detail_opens` — clicking People shows `[data-testid='people-page']`
- `test_people_detail_sections` — people detail contains section testids

### frontend/src/types.ts (PersonSummary added)
```typescript
export interface PersonSummary {
  path: string
  title: string
  updated_at: string
  open_actions: number
}
```
