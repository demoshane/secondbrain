## Decisions Made

- `sb_capture_batch` uses locally aliased imports (`_capture_note`, `_get_connection`, `_init_schema`, `_BRAIN_ROOT`) to avoid shadowing module-level names and keep the function self-contained
- `sb_tools` accesses `mcp._tool_manager._tools` directly (synchronous dict, no event loop) rather than calling `asyncio.run(mcp.list_tools())` which would raise `RuntimeError` inside FastMCP's stdio event loop
- `AttributeError` catch provides graceful fallback if FastMCP's private `_tool_manager` attribute changes in future versions
