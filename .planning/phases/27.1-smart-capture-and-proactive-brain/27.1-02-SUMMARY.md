## What Was Built

**engine/entities.py** — New module implementing `extract_entities(title, body) -> dict` using pure regex heuristics:
- `_extract_people()` — finds Title Case bigrams, filtered by `_STOP_WORDS`
- `_extract_topics()` — finds `#hashtag` patterns
- `_extract_places()` — finds capitalized tokens after location prepositions (in/at/from/to), excluding people tokens
- Title and body processed separately to prevent cross-boundary bigrams
- Outer try/except ensures the function never raises on any input

**engine/db.py** — Two additions:
- `get_connection(db_path=None)` — now accepts optional path for test isolation
- `migrate_add_entities_column(conn)` — idempotent `ALTER TABLE` migration
- `init_schema()` now calls `migrate_add_entities_column()` so new DBs get the column automatically

**engine/capture.py** — Three changes:
- `capture_note()` calls `extract_entities()` best-effort before `write_note_atomic()`, sets `post["entities"]`
- `write_note_atomic()` UPDATEs the entities column after INSERT, within the same transaction
- `main()` imports and calls `migrate_add_entities_column(conn)` for production DB upgrades

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed cross-boundary bigram extraction in extract_entities()**
- **Found during:** Task 2 verification
- **Issue:** Original implementation concatenated title and body with a space: `f"{title or ''} {body or ''}"`. For input title="Alice in Wonderland" + body="Alice Johnson visited London", this produced "Wonderland Alice" as a detected person name (end of title + start of body forming a spurious bigram).
- **Fix:** Process title and body separately in `_extract_people`, `_extract_topics`, `_extract_places`; merge results with set union. Eliminates all cross-boundary artifact matches.
- **Files modified:** engine/entities.py
- **Commit:** 1698284

**2. [Rule 2 - Missing functionality] Added optional db_path to get_connection()**
- **Found during:** Task 2 — test stub called `get_connection(str(tmp_path / "brain.db"))` but original signature took no args
- **Fix:** Added `db_path: str | None = None` parameter; defaults to existing `DB_PATH` when not provided — fully backward compatible
- **Files modified:** engine/db.py
- **Commit:** 1698284
