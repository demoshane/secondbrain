## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing (tag badges, action row internals) |
| md | 16px | Default element spacing (card padding, section content) |
| lg | 24px | Section padding (panel headers, page-level insets) |
| xl | 32px | Layout gaps (column gaps, modal padding) |
| 2xl | 48px | Major section breaks (page top padding) |
| 3xl | 64px | Page-level spacing (rarely used) |

Exceptions:
- Topbar height: 52px (observed from mockup proportions, not a multiple of 8 — use `h-[52px]`)
- TabBar height: 40px (use `h-10`)
- Sidebar width: 256px fixed (`w-64`) — per CONTEXT.md decision
- Right panel width: 256px fixed (`w-64`) — per design brief
- People list column width: 320px (`w-80`)
- Meetings/Projects list column width: 320px (`w-80`)
- Min-width touch target: 32px for icon buttons (desktop app, not mobile)

```css
  --card: 222 22% 11%;               /* #151b27 — sidebar, panels, cards */
  --popover: 222 24% 13%;            /* #181f2e — command palette, dropdowns */

  /* Foreground */
  --foreground: 215 25% 87%;         /* #d4dae8 — primary text */
  --card-foreground: 215 25% 87%;
  --popover-foreground: 215 25% 87%;

  /* Primary action (blue-indigo) */
  --primary: 231 87% 65%;            /* #5b6cf7 — primary buttons, active tab underline, links */
  --primary-foreground: 0 0% 100%;   /* #ffffff */

  /* Secondary surface */
  --secondary: 222 22% 16%;          /* #1e2535 — secondary buttons bg, hover states */
  --secondary-foreground: 215 25% 87%;

  /* Muted text and surfaces */
  --muted: 222 22% 14%;              /* #191f2d — muted backgrounds */
  --muted-foreground: 215 15% 52%;   /* #7a8499 — timestamps, metadata, muted labels */

  /* Accent (orange — Smart Capture, sparkles, AI features only) */
  --accent: 25 95% 53%;              /* #f97316 — Smart Capture button, AI sparkles icon, Brain Insight card */
  --accent-foreground: 0 0% 100%;

  /* Borders and inputs */
  --border: 222 20% 20%;             /* #2a3245 — dividers, card borders, table row borders */
  --input: 222 22% 14%;              /* #191f2d — input backgrounds */
  --ring: 231 87% 65%;               /* matches --primary for focus rings */

  /* Destructive */
  --destructive: 0 84% 60%;          /* #ef4444 — Delete buttons, error states */
  --destructive-foreground: 0 0% 100%;

  /* Radius */
  --radius: 0.375rem;                /* 6px — tighter than shadcn default 8px, per mockup */
}
```

## Interaction Contracts

### Tab bar
- Active tab: 2px bottom border in `--primary`, text weight 600, text color `--foreground`
- Inactive tab: text `--muted-foreground`, weight 400
- Hover: text `--foreground`, bg `--secondary` (subtle)
- Tab order: Notes · Actions · People · Meetings · Projects · Intelligence · Inbox · Links

### Tab icons (lucide-react)
| Tab | Icon |
|-----|------|
| Notes | `FileText` |
| Actions | `CheckSquare` |
| People | `Users` |
| Meetings | `Calendar` |
| Projects | `Briefcase` |
| Intelligence | `Brain` |
| Inbox | `Inbox` |
| Links | `Link` |

### Topbar buttons
- **New Note**: `Button` variant=`default`, size=`sm`, icon=`Plus`, label="New Note"
- **Smart Capture**: `Button` variant=`outline`, size=`sm`, icon=`Sparkles` (orange tint via `text-orange-400`), label="Smart Capture"
- **Batch**: `Button` variant=`ghost`, size=`sm`, icon=`FolderUp`, label="Batch"
- **Connection dot**: 8px circle, green-500 (connected) / amber-400 (reconnecting) / red-500 (disconnected); tooltip on hover

### Sidebar
- Collapsed section: chevron right, count badge right-aligned, `text-muted-foreground`
- Expanded section: chevron down, notes visible
- Active note row: `bg-secondary`, `text-foreground`, left 2px border in `--primary`
- Inactive note row: `text-muted-foreground`, hover `bg-secondary/50`
- Width: 256px fixed. No resize handle in Phase 41.
- Type grouping order: Meetings, People, Projects, Ideas, Notes, Strategy, Archive (frequency-based)

### Right Panel
- Collapsible via chevron on left edge
- Collapsed state: 40px wide icon strip
- Section hide when empty (do not show empty section headings)
- Full empty state only when ALL three sections are empty

### Command Palette
- Trigger: `Cmd+K`
- Overlay: `bg-black/60` backdrop
- Panel: max-w-[600px], bg `--popover`, border `--border`, radius `--radius`
- Selection highlight: `bg-secondary`
- Uses `cmdk` (already installed)

### Destructive actions
- Always require confirmation via `ConfirmDialog`
- Delete button: variant=`destructive`, hover-reveal (opacity-0 group-hover:opacity-100)
- Confirmation panel: red border, warning icon (`AlertTriangle`), names the thing being deleted

### Collapsible sections
- Smooth transition: `transition-all duration-200`
- State persisted to localStorage by `sectionId`
- Count badge: `bg-secondary text-muted-foreground` pill, shown on every section header

### Loading states
- List skeletons: `SkeletonList` component, matches row height of target list
- Inline spinners: `Loader2` icon, `animate-spin`, size 16px
- Never show blank area during async load

### Action item rows
- Checkbox: 20px, checked state strikethrough on text
- Done item: `text-muted-foreground line-through`
- Overdue date: `text-red-400`
- Assignee dropdown: compact, shows initials badge when assigned
- Delete: hover-reveal trash icon, opens `ConfirmDialog`

## Design Token Migration Instructions

For the executor implementing plan 41-01 (design tokens):

1. **Replace `frontend/src/index.css` entirely.** Delete the `.dark {}` block. Replace `:root {}` with the CSS var values in the Color section above. Keep `@tailwind base/components/utilities` directives.
2. **Update `frontend/tailwind.config.js`.** Set `darkMode: 'class'` → remove or change to `darkMode: false` (always dark — no class toggling needed). All `hsl(var(--*))` color references in the config remain unchanged; only the CSS var values change.
3. **Update `frontend/src/App.tsx`** (or wherever the root `div` has `className="dark"`): remove the `dark` class. The app is always dark.
4. **`--radius: 0.375rem`** replaces current `0.5rem`. All `rounded-lg` Tailwind classes will automatically use the tighter radius.
