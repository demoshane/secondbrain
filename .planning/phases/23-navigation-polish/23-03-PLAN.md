    **index.html changes:**
    1. Add `<div id="tag-chips" class="tag-chips-row"></div>` between #note-title and #note-content in the viewer panel.
    2. Add `<div id="filter-banner" style="display:none" class="filter-banner"></div>` above #note-list (or the search input, wherever the note list starts in the layout).

    **app.js changes:**

    Add module-level: `let activeTagFilter = null;` and `let _suppressNextTagRefresh = false;`

    **renderTagChips(tags, notePath)** — renders into #tag-chips:
    - Clear container innerHTML first
    - For each tag, create `<span class="tag-chip">` with textContent `#tagname`
    - Single-click on chip → activateTagFilter(tag) (tag filter, see Task 2)
    - Double-click on chip → makeChipEditable(chipEl, tagIndex, tags, notePath)
    - Append `<button class="tag-add-btn">+ Add tag</button>` at end; click → addNewTag(tags, notePath)
    - Use textContent (not innerHTML) for tag values to prevent XSS

    **makeChipEditable(chipEl, tagIndex, allTags, notePath)** — inline edit:
    - Create `<input type="text" class="tag-chip-input">` with value = allTags[tagIndex]
    - chipEl.replaceWith(input); input.focus(); input.select()
    - On Enter: commit (trim value, update allTags[tagIndex], call saveTags(updatedTags, notePath))
    - On Escape: re-render chips from current allTags (no save)
    - On blur: commit (same as Enter)

    **addNewTag(allTags, notePath)** — append a new blank chip-input at end, on commit add to allTags and saveTags

    **saveTags(tags, notePath)** — optimistic save:
    - Set _suppressNextTagRefresh = true
    - Immediately call renderTagChips(tags, notePath) (optimistic DOM update)
    - fetch PUT /notes/<encodedPath> with body {"tags": tags}
    - On failure: flash chip container (add class "tag-save-error" for 1.5s), re-render original tags, set _suppressNextTagRefresh = false

    **Update openNote()**: after rendering body, call `renderTagChips(note.tags || [], path)`. Note: GET /notes/<path> returns the raw file; tags must come from either the response metadata or the in-memory notes array. Check the existing /notes/<path> response — if it includes tags in the response already, use that; otherwise use the notes array from the last loadNotes() call (find by path).

    **Update handleNoteEvent()**: before calling openNote(currentPath) in the modified branch, check `if (_suppressNextTagRefresh) { _suppressNextTagRefresh = false; return; }` — skip the re-render for this one event.

  <files>engine/gui/static/app.js, engine/gui/static/style.css</files>
  <action>
    **app.js — tag filter state machine:**

    **activateTagFilter(tag)**:
    - Set activeTagFilter = tag
    - Show #filter-banner: innerHTML = `<span>Filtering: <strong>#${tag}</strong></span><button class="filter-clear-btn" id="filter-clear">×</button>`
    - Attach click on #filter-clear → clearTagFilter()
    - Call runTagFilter(tag)

    **clearTagFilter()**:
    - activeTagFilter = null
    - document.getElementById('filter-banner').style.display = 'none'
    - const q = document.getElementById('search-input').value.trim()
    - if (q) runSearch(q); else loadNotes()

    **runTagFilter(tag)**:
    - If there is an active search query in #search-input: call runSearch() (which now includes activeTagFilter — AND logic)
    - Otherwise: fetch all notes from the already-loaded notes array (module-level cache); filter client-side: notes.filter(n => (n.tags || []).includes(tag)); call renderFlatList(filtered)
    - Show filter banner (already shown by activateTagFilter)

    **Update runSearch()**:
    - When building the POST /search request body, if activeTagFilter !== null, add `tags: [activeTagFilter]` to the body
    - This provides AND logic: text query + tag filter

    **Update the search input's clear/empty handler**: when user clears search input back to empty string, if activeTagFilter is set, call runTagFilter(activeTagFilter) instead of loadNotes() — preserve tag filter

    **Update single-click on tag chips** (from Task 1): confirm activateTagFilter(tag) is wired on single-click, NOT double-click. Double-click = edit.
