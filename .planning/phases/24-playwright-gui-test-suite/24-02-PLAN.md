    Strategy: Use requests.put() directly against live_server_url to update the title,
    then wait for SSE to propagate the change to the open page. This avoids the EasyMDE
    async initialization timing issues identified in research (Pitfall 3).

    Implementation:
    ```python
    def test_title_sync(page, live_server_url, gui_brain, seed_note_fn):
        import requests
        note_path = seed_note_fn(gui_brain, "Original Title", "body content")

        page.goto("/ui")
        page.wait_for_selector("#sidebar-loading", state="hidden", timeout=5000)

        # Click the note to open it
        page.locator("#note-list li[data-path]", has_text="Original Title").click()
        page.locator("#viewer").wait_for(state="visible", timeout=5000)

        # Update title via API — triggers SSE broadcast
        import urllib.parse
        encoded = urllib.parse.quote(note_path, safe="")
        resp = requests.put(
            f"{live_server_url}/notes/{encoded}",
            json={"title": "Updated Title", "body": "body content"},
        )
        assert resp.status_code == 200

        # SSE broadcasts note_updated → sidebar and viewer heading update
        # Wait for sidebar li to reflect new title (up to 3s)
        page.wait_for_selector(
            "#note-list li[data-path]",
            has_text="Updated Title",
            timeout=3000,
        )
        # Viewer heading also updates
        page.locator("#viewer h1", has_text="Updated Title").wait_for(timeout=3000)
    ```

    Note: The `note_path` from seed_note_fn is an absolute path like
    /private/var/folders/.../ideas/original-title.md — it must be URL-encoded when
    used as a route segment. Use urllib.parse.quote(note_path, safe="") to encode slashes.
    The _SlashNormMiddleware in api.py handles the double-slash from /notes//abs/path.
    Actually, simpler: pass the path directly since Flask's <path:note_path> converter
    accepts slashes. URL-encode only to avoid Flask routing issues. Check how test_api.py
    handles absolute paths in URL — use the same pattern.
