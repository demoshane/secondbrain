## Interaction Contract

### AI Provider section layout (inside SettingsModal, new section block)

Section heading: `"AI Provider"` ŌĆö matches existing heading style (`text-sm font-medium mb-3`).

Subsection A ŌĆö Groq API Key:
```
Label: "Groq API key"
Description: "Store your Groq API key in macOS Keychain for fast AI responses (<20s)."

State: unconfigured
  - password Input (h-8, placeholder "gsk_ŌĆ”", type="password")
  - "Save key" Button (size="sm") ŌĆö disabled when input empty
  - No status indicator

State: configured (key present in Keychain)
  - Input hidden
  - Green "Ō£ō Configured" text (text-xs text-green-500, flex items-center gap-1, CheckCircle2 h-3 w-3)
  - Connectivity result inline: "Ō£ō Connected" (green) or "Ō£Ś Invalid key" (text-destructive)
  - "Remove key" Button (size="sm", variant="outline") ŌĆö triggers DELETE /config/groq, returns to unconfigured state
  - No confirmation dialog required for Remove key (non-destructive, key can be re-entered)

Connectivity test:
  - Runs automatically after successful Save (POST /config/groq/test)
  - Shows Loader2 spinner next to "Configured" text while test in flight
  - Result replaces spinner inline ŌĆö never a separate step
```

Subsection B ŌĆö All-local toggle:
```

  Label: "All-local mode" (text-xs font-medium)
  Description: "Use local Ollama for everything ŌĆö full privacy mode" (text-xs text-muted-foreground)
Right: Switch component (checked = all_local = true)

When ON: all four Groq feature toggles below become opacity-50 pointer-events-none (visually disabled)
When OFF: feature toggles active
```

Subsection C ŌĆö Groq feature toggles (visible only when Groq key is configured):
```

  Label: "Ask Brain"

  Label: "Follow-up questions"
  Description: "Generate follow-up prompts via Groq"
```

### Fallback toast

When Ask Brain response returns `"provider": "fallback"`:
- Toast message: `"Groq unavailable ŌĆö used fallback model"`
- Variant: default (not destructive) ŌĆö amber tint via `text-amber-400` in toast description
- Duration: 4 seconds
- Trigger: in `AskBrainModal.tsx` after response, not in SettingsModal

### Save flow

The existing global Save button at the bottom of SettingsModal saves routing + ollama + markers. The Groq API key is saved independently via its own "Save key" button (calls POST /config/groq directly). The all-local toggle and feature toggles are saved via the global Save button (they write to config.toml).
