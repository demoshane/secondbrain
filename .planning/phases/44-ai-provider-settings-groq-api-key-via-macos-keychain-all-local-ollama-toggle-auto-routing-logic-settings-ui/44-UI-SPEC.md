## Interaction Contract

### AI Provider section layout (inside SettingsModal, new section block)

Section heading: `"AI Provider"` — matches existing heading style (`text-sm font-semibold mb-3`).

Subsection A — Groq API Key:
```
Label: "Groq API key"
Description: "Store your Groq API key in macOS Keychain for fast AI responses (<20s)."

State: unconfigured
  - password Input (h-8, placeholder "gsk_…", type="password")
  - "Save key" Button (size="sm") — disabled when input empty
  - No status indicator

State: configured (key present in Keychain)
  - Input hidden
  - Green "✓ Configured" text (text-xs text-green-500, flex items-center gap-1, CheckCircle2 h-3 w-3)
  - Connectivity result inline: "✓ Connected" (green) or "✗ Invalid key" (text-destructive)
  - "Remove" Button (size="sm", variant="outline") — triggers DELETE /config/groq, returns to unconfigured state
  - No confirmation dialog required for Remove (non-destructive, key can be re-entered)

Connectivity test:
  - Runs automatically after successful Save (POST /config/groq/test)
  - Shows Loader2 spinner next to "Configured" text while test in flight
  - Result replaces spinner inline — never a separate step
```

Subsection B — All-local toggle:
```
Layout: flex items-center justify-between py-2
Left: label + description stacked
  Label: "All-local mode" (text-xs font-medium)
  Description: "Use local Ollama for everything — full privacy mode" (text-xs text-muted-foreground)
Right: Switch component (checked = all_local = true)

When ON: all four Groq feature toggles below become opacity-50 pointer-events-none (visually disabled)
When OFF: feature toggles active
```

Subsection C — Groq feature toggles (visible only when Groq key is configured):
```
Four rows, each: flex items-center justify-between py-1.5
Labels and descriptions:

ask_brain:
  Label: "Ask Brain"
  Description: "Route brain queries to Groq"

followup_questions:
  Label: "Follow-up questions"
  Description: "Generate follow-up prompts via Groq"

digest:
  Label: "Weekly digest"
  Description: "Synthesise your digest using Groq"

person_synthesis:
  Label: "Person insights"
  Description: "Generate people context via Groq"

Disabled appearance when all-local ON: opacity-50, Switch disabled prop true
Hidden entirely when Groq key is NOT configured (no point showing toggles with no key)
```

### Fallback toast

When Ask Brain response returns `"provider": "fallback"`:
- Toast message: `"Groq unavailable — used fallback model"`
- Variant: default (not destructive) — amber tint via `text-amber-400` in toast description
- Duration: 4 seconds
- Trigger: in `AskBrainModal.tsx` after response, not in SettingsModal

### Save flow

The existing global Save button at the bottom of SettingsModal saves routing + ollama + markers. The Groq API key is saved independently via its own "Save key" button (calls POST /config/groq directly). The all-local toggle and feature toggles are saved via the global Save button (they write to config.toml).
