## Decisions Made

- `[data-testid="note-body"]` is the correct target for viewer scroll test — this is the scrollable content div inside NoteViewer

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed vite base URL causing asset 404s**
- **Found during:** Task 2 (human verification)
- **Issue:** `base: './'` in vite.config.ts produced relative asset paths; when loaded from `/ui/`, paths resolved incorrectly and all JS/CSS assets returned 404
- **Fix:** Changed `base` to `'/ui/'` and rebuilt the bundle
- **Files modified:** `frontend/vite.config.ts`, `engine/gui/static/index.html`, rebuilt assets
- **Verification:** React app loads correctly; window.API_BASE confirmed in devtools
- **Committed in:** `51b35e1`

**2. [Rule 1 - Bug] Fixed SSEContext not receiving SSE events**
- **Found during:** Task 2 (human verification)
- **Issue:** `src.onmessage` only fires for unnamed SSE events; Flask `_broadcast()` emits `event: note` typed events which require `addEventListener('note', ...)`
- **Fix:** Replaced `src.onmessage` with `src.addEventListener('note', handler)`; also added `loadNotes()` call on `onopen` to populate sidebar on initial connect
- **Files modified:** `frontend/src/contexts/SSEContext.tsx`
- **Verification:** SSE status dot green; sidebar populates on load; 9 Playwright tests pass
- **Committed in:** `51b35e1`

**3. [Rule 2 - Missing] Extended GET /notes/<path> to return title/type/tags**
- **Found during:** Task 2 (human verification — inline tag editing needed these fields)
- **Issue:** API only returned `body` and `path`; NoteViewer needed `title`, `type`, `tags` to reconstruct frontmatter for inline tag saves
- **Fix:** Extended response with `meta.get('title')`, `meta.get('type')`, `meta.get('tags')` with JSON parse fallback
- **Files modified:** `engine/api.py`
- **Verification:** NoteViewer inline tag editing saves correctly
- **Committed in:** `51b35e1`

**4. [Rule 2 - Missing] Added inline tag editing in NoteViewer**
- **Found during:** Task 2 (human verification)
- **Issue:** Tag chips were click-to-filter only; no way to edit tags inline as specified in plan (dblclick interaction implied by test coverage)
- **Fix:** Added `localTags` state, `editingTag` state, dblclick handler on Badge, inline input on Enter/Escape with `saveNote` call
- **Files modified:** `frontend/src/components/NoteViewer.tsx`
- **Verification:** Playwright tag tests pass; dblclick enters edit mode
- **Committed in:** `51b35e1`
