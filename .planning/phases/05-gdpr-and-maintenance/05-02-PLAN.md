  <behavior>
    - test_pii_note_denied_no_passphrase: SB_PII_PASSPHRASE unset, getpass.getpass mocked to raise EOFError → returns 1, no note content printed
    - test_pii_note_denied_wrong_passphrase: SB_PII_PASSPHRASE="correct", SB_PII_PASSPHRASE_INPUT="wrong" → returns 1
    - test_pii_note_shown_correct_passphrase: SB_PII_PASSPHRASE="correct", SB_PII_PASSPHRASE_INPUT="correct" → returns 0
    - test_non_pii_note_no_gate: note with content_sensitivity: public, no env vars set → returns 0 without prompting
  </behavior>
  <implementation>
Implement read_note(path, conn) as described in RESEARCH.md Pattern 2:

1. Check path.exists(); if not, print error to stderr (never include user-supplied path content), return 1.
2. frontmatter.load(str(path)) to get post. On exception, print type(e).__name__ to stderr, return 1.
3. sensitivity = post.get("content_sensitivity", "public").
4. If sensitivity == "pii":
   a. expected = os.environ.get("SB_PII_PASSPHRASE", "").
   b. If expected is empty string: print "Access denied: passphrase required for PII note." return 1.
   c. entered = os.environ.get("SB_PII_PASSPHRASE_INPUT", "").
   d. If entered is empty: call getpass.getpass("Passphrase: ") inside try/except (EOFError, KeyboardInterrupt) → on exception print denial message, return 1.
   e. If entered != expected: print denial message, return 1.
5. Log audit: INSERT INTO audit_log (event_type, note_path, created_at) VALUES ('read', str(path), utcnow). Wrap in try/except — audit is best-effort, never blocks read (consistent with search.py pattern).
6. Print frontmatter header block (---, title, type, sensitivity, ---) then post.content.
7. Return 0.

Implement main() with argparse: positional arg "path", calls get_connection() + init_schema() + read_note() + sys.exit(code).

Denial message constant: _ACCESS_DENIED = "Access denied: passphrase required for PII note."
Never print note content in error paths (GDPR-05).
  </implementation>
</feature>
