**Step 2 — Add Task 3 block inside `_run_intelligence_hooks()` (after the action items try/except block, before the function ends):**

```python
        if not _skip_stubs:
            try:
                from engine.db import get_connection as _get_conn
                from engine.segmenter import resolve_entities
                _conn3 = _get_conn()
                try:
                    resolution = resolve_entities(_entities, _conn3, _brain_root)
                    for stub in resolution.get("new_stubs", []):
                        try:
                            capture_note(
                                note_type=stub["type"],
                                title=stub["name"],
                                body="",
                                tags=[],
                                people=[],
                                content_sensitivity="public",
                                brain_root=_brain_root,
                                conn=_conn3,
                            )
                            _conn3.commit()
                        except Exception:
                            pass
                finally:
                    _conn3.close()
            except Exception:
                pass
```

Per D-02: runs async in existing daemon thread. Per D-03: no response surface changes.

**Key details:**
- `_conn3` is a fresh connection via `get_connection()` — NOT the caller's conn (Pitfall 1 from RESEARCH.md)
- `capture_note()` recursive call uses `_conn3` — they serialize naturally within the single thread
- `_conn3.commit()` after each stub — same pattern as sb_capture_smart
- Triple try/except nesting: outer catches import/conn errors, inner catches per-stub errors — silent-catch pattern matching existing hooks
- `_brain_root` is a `Path` (already a closure variable) — `resolve_entities` needs `Path`, not `str`

**No other changes.** No changes to function signature, return value, or any other code path.
