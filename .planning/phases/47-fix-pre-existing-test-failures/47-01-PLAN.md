The fixture sets `brain_root = tmp_path / "brain"` but does NOT resolve the symlink. On macOS, `tmp_path` is `/tmp/pytest-xxx/...` which is a symlink to `/private/tmp/pytest-xxx/...`. When `capture_note()` internally calls `target.resolve()` via `store_path()`, the notes table gets paths relative to `/private/tmp/...`, but the test fixture's BRAIN_ROOT is `/tmp/...`. FK lookups then fail.

Fix: resolve `brain_root` after creation:
```python
brain_root = tmp_path / "brain"
brain_root.mkdir()
brain_root = brain_root.resolve()  # <-- add this line
```
This makes BRAIN_ROOT = `/private/tmp/.../brain`, matching what `store_path()` produces internally.
