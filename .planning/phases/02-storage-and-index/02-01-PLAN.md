  <behavior>

    - build_post(note_type, title, body, tags, people, content_sensitivity): returns frontmatter.Post with all 8 fields (type, title, date, tags, people, created_at, updated_at, content_sensitivity)
    - write_note_atomic(target: Path, post: frontmatter.Post, conn): temp file in same dir via tempfile.mkstemp(dir=target.parent); INSERT into notes + log_audit in one conn.commit(); then os.replace; delete temp on any exception
    - log_audit(conn, event_type, note_path): INSERT into audit_log; caller commits
    - On exception: error message is f"Failed to write {target}: {type(e).__name__}" — NEVER includes post.content or post.metadata values
    - render_template(template_str, substitutions): uses string.Template.safe_substitute()
    - load_template(note_type): loads TEMPLATES_DIR / f"{note_type}.md"; returns raw string; creates minimal fallback if file missing
  </behavior>
  <action>
1. In engine/db.py: add migrate_add_people_column(conn) function. Call it at end of init_schema() (after executescript). This ensures any existing brain.db from Phase 1 gets the people column automatically.

2. Create engine/templates.py:
   - load_template(note_type: str) -> str: reads TEMPLATES_DIR / f"{note_type}.md". If file not found, returns a minimal fallback template string with ${title} and ${date} placeholders. Uses pathlib only (FOUND-12).
   - render_template(template_str: str, substitutions: dict) -> str: returns string.Template(template_str).safe_substitute(substitutions). Never raises on missing keys (safe_substitute leaves them as-is).

3. Create engine/capture.py with these functions:
   - build_post(note_type, title, body, tags, people, content_sensitivity="public") -> frontmatter.Post: sets all 8 required fields. date = datetime.date.today().isoformat(), created_at/updated_at = datetime.datetime.utcnow().strftime("%Y-%m-%dT%H:%M:%SZ").
   - log_audit(conn, event_type: str, note_path: str) -> None: INSERT INTO audit_log (event_type, note_path, created_at). Caller is responsible for conn.commit().
   - write_note_atomic(target: Path, post: frontmatter.Post, conn: sqlite3.Connection) -> None: implements two-phase write. Exact order: (1) mkstemp(dir=target.parent), (2) write frontmatter.dumps(post) to fd, (3) INSERT notes row + log_audit("create", ...) + conn.commit(), (4) os.replace(tmp, target). On any exception: unlink tmp if it exists, re-raise. tags and people stored as JSON text (json.dumps(list)).
   - capture_note(note_type, title, body, tags, people, content_sensitivity, brain_root, conn) -> Path: builds target path as brain_root / note_type / f"{datetime.date.today().isoformat()}-{title[:40].replace(' ', '-').lower()}.md", ensures parent exists, calls write_note_atomic. Returns final path.

4. notes INSERT in write_note_atomic: INSERT INTO notes (path, type, title, body, tags, people, created_at, updated_at, sensitivity) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?). Use str(target) as path (relative-or-absolute consistent with caller). The sensitivity column maps from content_sensitivity field.

CRITICAL: never use `/tmp` for tempfile — always pass dir=target.parent to mkstemp.
CRITICAL: conn.commit() must happen BEFORE os.replace() — see RESEARCH.md Pitfall 2.
