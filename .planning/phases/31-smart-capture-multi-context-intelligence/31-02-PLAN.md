<task type="auto" tdd="true">
  <name>Task 2: Three-path dedup heuristic for near-duplicate segments</name>
  <files>engine/segmenter.py, engine/mcp_server.py, tests/test_smart_capture.py</files>
  <behavior>
    - test_dedup_superset: new body longer and contains existing key phrases -> updates existing note, appends changelog
    - test_dedup_complementary: different angle on same topic -> saves as new + creates 'similar' relationship
    - test_dedup_ambiguous: cannot determine -> returns both options without saving that segment
    - test_dedup_no_match: no similar notes -> saves normally
    - test_changelog_format: updated note has ## Changelog section with date and previous hash
  </behavior>
  <action>
    Add `dedup_segment(title: str, body: str, conn, brain_root: Path) -> dict` to `engine/segmenter.py`:

    1. Call `check_capture_dedup(title, body, conn)` — returns list of similar notes or []
    2. If empty: return `{"action": "save_new"}`
    3. If matches found, take top match:
       a. Read existing note body from DB: `conn.execute("SELECT body FROM notes WHERE path=?", (match["path"],)).fetchone()`
       b. **Superset check**: `len(body) > len(existing_body) * 1.2` AND at least 2 of first 5 "key phrases" (words >4 chars from existing body[:200]) found in new body
          - Return `{"action": "update_existing", "path": match["path"], "existing_body": existing_body}`
       c. **Complementary**: not superset -> `{"action": "save_complementary", "similar_path": match["path"]}`
       d. **Ambiguous** (3+ matches all above threshold): `{"action": "ambiguous", "options": matches}`

    Wire into `sb_capture_smart`:
    - Before saving each segment, call `dedup_segment()`
    - "update_existing": read existing note, append new content + `## Changelog` section with `hashlib.sha256(existing_body.encode()).hexdigest()[:8]`, write back via `write_note_atomic(path, updated_post, conn, update=True)`
    - "save_complementary": save new note normally, then INSERT `similar` relationship
    - "ambiguous": skip saving, add to response `ambiguous_segments` list with options for caller to decide
    - "save_new": normal save via `capture_note()`

    Promote `test_dedup_three_path` and `test_multi_context_atomic_save` xfail stubs to real tests.
  </action>

  </verify>
