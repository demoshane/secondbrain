## Integration Findings

### Critical Issues (require fix before v1.0 sign-off)

**Issue 1 — Path format split (SEARCH-01, SEARCH-04, GDPR-01)**

`engine/capture.py:write_note_atomic()` stores DB paths relative to `BRAIN_ROOT` (e.g. `notes/2026-slug.md`). `engine/reindex.py` stores absolute paths (e.g. `/workspace/brain/notes/2026-slug.md`). Until `sb-reindex` is run:
- `engine/rag.py:retrieve_context()` raises `FileNotFoundError` for newly captured notes
- `engine/forget.py:forget_person()` silently skips DB rows for capture-stored relative paths

**Fix location:** `engine/capture.py` in `write_note_atomic()` — change to `str(target.resolve())` for the path stored in DB.

### Low-Severity Issues (tech debt)

**Issue 2 — CAP-06/AI-05 routing bypass**
`engine/ai.py:update_memory()` accepts `config_path` but always uses `ClaudeAdapter()` directly. User's model routing config has no effect on memory updates. Dead parameter. Fix: wire through `get_adapter("public", config_path)` or remove the dead parameter.

**Issue 3 — FOUND-08/GDPR-06 Anthropic key gap (documented)**
`detect-secrets` v1.5.0 has no coverage for `sk-ant-api03-*` Anthropic key patterns. Documented in Phase 1 plan 01-07. Not a regression.
