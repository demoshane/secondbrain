### 2. Stale docstring — engine/ai.py:126

`update_memory()` docstring says *"Always uses ClaudeAdapter regardless of sensitivity"*. Phase 08 fixed the implementation at line 141 but the docstring was not updated.

**Fix:** Update to *"Uses _router.get_adapter('public', config_path) — adapter driven by config.toml public_model."*

### 3. GDPR Requirement Definition Ambiguity

Three GDPR requirements were internally scoped narrower than standard GDPR meaning:

- **GDPR-02** (data portability): Implemented as FTS5 rebuild after forget. No `sb-export` CLI.
- **GDPR-06** (anonymization): Implemented as detect-secrets pre-commit check. No runtime `anonymize()`.
- **GDPR-03** (consent): Implemented as comprehensive audit logging. No first-run consent prompt.

**Action:** Decide if v1.5 intentionally excludes these or if gap phases are needed before production use.

### 4. forget.py path construction (latent test risk)

`forget_person()` constructs deletion paths as `str(brain_root / "people" / slug)` without `.resolve()`. Production-safe (`~/SecondBrain` resolves cleanly). Risk only in test environments using `tmp_path` with symlinked `/tmp`.

**Fix:** Add `.resolve()` to `brain_root` at `forget_person()` entry for consistency with Phase 07 fix.

### 5. Human Verification Pending (4 phases)
