## Critical Gaps (Blockers)

### GAP-1: CAP-06 — update_memory() never called

**Severity:** Critical (requirement completely unimplemented at integration level)
**File:** `engine/ai.py` — function exists at line 122; `engine/capture.py` — never imports or calls it
**Fix:** Add `update_memory(args.note_type, result_summary)` call in `capture.py:main()` after `capture_note()` returns, but only for non-PII notes.

### GAP-2: AI-02 / AI-03 — Watcher bypasses PII classifier

**Severity:** Critical (security/privacy boundary violation for file watcher path)
**File:** `engine/watcher.py:102` — `get_adapter("private", CONFIG_PATH)` hardcoded at daemon startup
**Fix:** Move `get_adapter()` call inside `on_new_file()`, after calling `classifier.classify(note_type, file_content)` on the dropped file's text. Adapter must be resolved per-file, not per-daemon.

### GAP-3: Path format mismatch — capture vs reindex

**Severity:** Critical (silent data corruption — RAG returns "[note file not readable]" after reindex)
**File:** `engine/reindex.py:43` — `str(md_path.relative_to(brain_root))` vs `engine/capture.py:109` — `str(target)` (absolute)
**Fix:** In `reindex.py:43`, replace `md_path.relative_to(brain_root)` with `md_path` to store absolute paths, matching `capture.py`.
